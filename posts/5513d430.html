算法和计算复杂度 5 The Polynomoial-Time Hierarchy

Created2024-07-12|Updated2024-07-12|AUDatalogi算法和计算复杂度
|Word Count:1.7k|Reading Time:10mins|Post Views:

Idea: arithmetic hierarchy from recursion theory.

Halting problem HALTHALTHALT is a recursive enumerable language that is undecidable.

The polynomial-time hierarchy considers polynomial time and SATSATSAT instead of HALTHALTHALT.

Oracle Turing Machines

An oracle Turing machine M?M^?M? is equipped with an additional tape, the query tape, and has three new special states qask,qyes,qnoq_{ask},q_{yes},q_{no}qask​,qyes​,qno​. The query tape is a write-only tape.

The operation of M?M^?M? is defined relative to an oracle language AAA.

Denote M?M^?M? together with oracle language AAA by MAM^AMA.

Whenever MAM^AMA enters the state qaskq_{ask}qask​:

  • Let zzz be the string of symbols written to the query tape. The query tape is reinitialized to a black tape.
  • If z∈Az\in Az∈A, computation resumes in the state qyesq_{yes}qyes​.
  • If z∉Az\not\in Az∈A, computation resumes in the state qnoq_{no}qno​.

Define timeMA(x)time_{M^A}(x)timeMA​(x) and spaceMA(x)space_{M^A}(x)spaceMA​(x) for an oracle Turing machine with an oracle language AAA to be the number of computation steps and number of cells of work tape used during computation on input xxx. The same way we define time-bounded and space-bounded oracle Turing machines.

Then we need to define oracle versions of complexity classes. Straightforward to define time-bounded complexity classes, but be careful to define space-bounded complexity classes.

DTIMEA(T(n)),DSPACEA(S(n)),NTIMEA(T(n))DTIME^A(T(n)),DSPACE^A(S(n)),NTIME^A(T(n))DTIMEA(T(n)),DSPACEA(S(n)),NTIMEA(T(n)).

❗Problem for non-deterministic space-bounded oracle Turing machine: it may write very long strings to the oracle tape.

Solution: Requiring the oracle Turing machine to behave deterministically in the time between writing the first symbol to the oracle tape and making the oracle query. (deterministic oracle access)

Thus we define NSPACEA(S(n))NSPACE^A(S(n))NSPACEA(S(n)) as the class of languages computed by a O(S(n))O(S(n))O(S(n)) space-bounded non-deterministic oracle Turing machine with deterministic oracle access to oracle AAA.

The definition can be extended to other complexity classes.

PAP^APA and NPANP^ANPA for any oracle language AAA.

The oracle may be chosen from some class of languages. Let C\mathcal CC be a class of languages.

DTIMEC(T(n))=⋃A∈CDTIMEA(T(n))DTIME^\mathcal C(T(n))=\bigcup_{A\in\mathcal C}DTIME^A(T(n))DTIMEC(T(n))=A∈C⋃​DTIMEA(T(n))

Similarly we have DSPACEC(T(n))DSPACE^\mathcal C(T(n))DSPACEC(T(n)), NTIMEC(T(n))NTIME^\mathcal C(T(n))NTIMEC(T(n)) and NSPACEC(T(n))NSPACE^\mathcal C(T(n))NSPACEC(T(n)).

This again extends to all other complexity classes such as PNPP^{NP}PNP.

Turing Reductions

coC={L‾∣L∈C}co\mathcal C=\{\overline L|L\in\mathcal C\}coC={L∣L∈C}

The Karp-Lipton Theorem

A possible way to prove P≠NPP\neq NPP=NP would be to show that there is some language in NPNPNP that is not computed by a family of polynomial size Boolean circuits, i.e., that NP⊈P/polyNP\not\subseteq P/polyNP⊆P/poly.

This is stronger than P≠NPP\neq NPP=NP since P⊆P/polyP\subseteq P/polyP⊆P/poly. It turns out that it is not stronger than the assumption that the polynomial-time hierarchy does not collapse.

🎯Theorem 32. Suppose that NP⊆P/polyNP\subseteq P/polyNP⊆P/poly. Then PH=Σ2P∩Π2PPH=\Sigma_2^P\cap\Pi_2^PPH=Σ2P​∩Π2P​.

Proof by Theorem 29, Corollary 4.
