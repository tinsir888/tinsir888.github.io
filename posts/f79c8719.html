算法和计算复杂度 6 Randomized Computation

What if Turing machine models have access to a source of randomness?

This would allow to compute more in same time/space, at the price of possibly not giving correct answer with small probability.

Current consensus: Randomization only gives

  • Polynomial saving in time
  • Constant factor saving in space

Because of hardness-based pseudo-random generator.

Exponential time: E=DTIME(2O(n))=UL−SIZE(2O(n))E=DTIME(2^{O(n)})=U_L-SIZE(2^{O(n)})E=DTIME(2O(n))=UL​−SIZE(2O(n)). (By corollary 2)

Recall that SIZE(2n/n)SIZE(2^n/n)SIZE(2n/n) includes every Boolean function on nnn variables, making non-uniform inclusion E⊆SIZE(2O(n))E\subseteq SIZE(2^{O(n)})E⊆SIZE(2O(n)).

Corresponding randomized complexity class BPPBPPBPP.

Theorem 33 (Impagliazzo and Wigderson). If there exists L∈EL\in EL∈E s.t. for every nnn and every Boolean circuit CCC computing LLL on input length nnn, the size of CCC must be 2Ω(n)2^{\Omega(n)}2Ω(n), then P=BPPP=BPPP=BPP.

Probabilistic Turing Machines

Give a probabilistic Turing machine an extra read-only and read-once tape which is pre-initialized with uniform and independent random bits. The tape head is allow to only stop or move forward.

Thus everything related to the Turing machines becomes a random variable.

Different kinds of Errors

Zero-sided error aka Las Vegas computation: Turing machine halt in a state “?” with small probability and require correct answer otherwise.

One-sided error: Turing machine may produce false negatives.

Two-sided error: Turing machine may produce false negatives and false positives.

Success Amplification

Error probability of 1/41/41/4 is large, but we can reduce it by independently running multiple times.

For zero-sided error, we won’t stop until we get non-unknown answer.

For one-sided error, if some trial accepts, we accept. otherwise we reject.

For two-sided error, we decided the answer by the majority of trials.

PIT

Polynomial Identity Testing (PIT): Testing whether a polynomial is identically zero.

PIT problem is efficiently solvable in randomized polynomial time but not known to be solvable deterministically in polynomial time.

AFIT

aka Arithmetic formula identity testing

A variant of PIT: input polynomial is given by an arithmetic formula.

Instance: Arithmetic formula computing the polynomial P(x1,⋯ ,xn)∈Z[x1,⋯ ,xn]P(x_1,\cdots,x_n)\in\mathbb Z[x_1,\cdots,x_n]P(x1​,⋯,xn​)∈Z[x1​,⋯,xn​].

Question: Is P(x)≡0P(x)\equiv0P(x)≡0?

A simple randomized algorithm: simply choose number a1,⋯ ,an∈{1,2,⋯ ,4m}a_1,\cdots,a_n\in\{1,2,\cdots,4m\}a1​,⋯,an​∈{1,2,⋯,4m} independently and uniformly at random, where mmm is the size of the given formula, evaluate P(a1,⋯ ,an)P(a_1,\cdots, a_n)P(a1​,⋯,an​) and answer accordingly.

Using Schwartz-Zippel lemma to analyze error probability.

Let F\mathbb FF be a field, let P(x1,⋯ ,xn)∈F[x1,⋯ ,xn]P(x_1,\cdots, x_n)\in\mathbb F[x_1,\cdots,x_n]P(x1​,⋯,xn​)∈F[x1​,⋯,xn​] be a non-zero polynomial of total degree at most ddd. ACIT

aka arithmetic circuit identity testing

Instance: Arithmetic circuit computing the polynomial P(x1,⋯ ,xn)∈Z[x1,⋯ ,xn]P(x_1,\cdots,x_n)\in\mathbb Z[x_1,\cdots,x_n]P(x1​,⋯,xn​)∈Z[x1​,⋯,xn​].

Question: Is P(x)≡0P(x)\equiv0P(x)≡0?

Circuit of size mmm has at most mmm multiplication gates. The degree of the polynomial computed is at most 2m2^m2m. The same approach as in arithmetic formula calls for draw random number from 1,⋯ ,2m+21,\cdots,2^{m+2}1,⋯,2m+2. (exponential!)

To remedy this, we switch to modular arithmetic with randomly chosen prime modulus. Use prime number theorem to analyze error probability.

We have alternative characterization of BPPBPPBPP. Also there are similar alternatives of ZPPZPPZPP and RPRPRP.

Randomized Computation and the Polynomial-time Hierarchy

By definition, RP⊆NPRP\subseteq NPRP⊆NP.

But we don’t know whether BPP⊆NPBPP\subseteq NPBPP⊆NP or NP⊆BPPNP\subseteq BPPNP⊆BPP.

We just guess: BPP=PBPP=PBPP=P thus BPP⊆NPBPP\subseteq NPBPP⊆NP.

But we can prove that BPPBPPBPP is contained in the second level of polynomial-time hierarchy.
