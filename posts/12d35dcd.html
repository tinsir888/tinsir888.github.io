Fair AI/ML 2 PROP Fair Clustering

Created2024-10-04|Updated2024-10-25|AUDatalogi算法博弈论AI/ML 公平性

Read the paper Proportionally Fair Clustering by Chen et al.

Abstract

nnn points, kkk centers.

Proportionality: any n/kn/kn/k points are entitled to form own cluster if there is another center that is closer in distance for all n/kn/kn/k points.

Clustering with no justified complaints from any subset of agents.

Tradeoff between proportional solutions and kkk-means objective.

Introduction

In centroid clustering, we want to partition data into kkk clusters by choosing kkk centers and then matching points to one of the centers.

Each data point represents a individual. We with the clustering to be fair. These agents prefer to be clustered accurately.

A clustering is fair if it respects the entitlements of groups of agents: a subset of agents is entitled to choose a center for themselves if they constitute a sufficiently large fraction of the population w.r.t. the total number of clusters. No priori knowledge about which points should be protected.

Preliminaries

N\mathcal NN data points, ∣N∣=n|\mathcal N|=n∣N∣=n.

M\mathcal MM feasible cluster centers, ∣M∣=m|\mathcal M|=m∣M∣=m.

For i,j∈M∪Ni,j\in\mathcal M\cup\mathcal Ni,j∈M∪N, we have distance (i,j)(i,j)(i,j) satisfying the triangle inequality.

Open a set X⊆MX\subseteq\mathcal MX⊆M of ∣X∣=k|X|=k∣X∣=k centers, and then match all points in N\mathcal NN to the closest center in XXX.

For a particular solution XXX and agent i∈Ni\in\mathcal Ni∈N, let Di(X)=min⁡x∈Xd(i,x)D_i(X)=\min_{x\in X}d(i,x)Di​(X)=minx∈X​d(i,x). A good clustering solution XXX will have small values of Di(X)D_i(X)Di​(X).

Different measures, different objectives:

  • kkk-median: ∑i∈NDi(X)\sum_{i\in\mathcal N}D_i(X)∑i∈N​Di​(X)
  • kkk-means: ∑i∈N(Di(X))2\sum_{i\in\mathcal N}(D_i(X))^2∑i∈N​(Di​(X))2

Individuals prefer to be closer to their center in terms of distance.

❗The quantification is over all subset of sufficient size. Do not consider a single i∈Si\in Si∈S and ignore all of the other points.

Advantages of PROP

  • PROP implies weak Pareto optimality: For any PROP solution XXX, there does not exist another solution X′X'X′ s.t. Di(X′)<Di(X)D_i(X')\lt D_i(X)Di​(X′)<Di​(X) for all i∈Ni\in\mathcal Ni∈N.
  • Oblivious, does not depend on sensitive attribute or protected sub-groups.
  • Robust to outlier.
  • Scale invariant: a multiplicative scaling of all distances does not affect the set of PROP solution.
  • Approximately PROP can be efficiently computed.
  • Can be efficiently checked: no need to compute entire pairwise distance matrix.

However, PROP is incompatible with kkk-center, kkk-means, kkk-median in the worst case.

X⊆MX\subseteq\mathcal MX⊆M with ∣X∣=k|X|=k∣X∣=k is ρ\rhoρ-approximate proportional (abbr. ρ\rhoρ-proportional) if ∀S⊆N\forall S\subseteq\mathcal N∀S⊆N with ∣S∣>⌈nk⌉|S|\gt\lceil\frac{n}{k}\rceil∣S∣>⌈kn​⌉ and ∀y∈M\forall y\in\mathcal M∀y∈M, there exists i∈Si\in Si∈S with ρ⋅d(i,y)≥Di(X)\rho\cdot d(i,y)\ge D_i(X)ρ⋅d(i,y)≥Di​(X).

Local Capture Heuristic

Local Capture heuristic produces more proportional clustering.

It takes a target value of ρ\rhoρ as a parameter, and proceeds by iteratively finding a center that violates ρ\rhoρ-fairness and swapping it for the center in the current solution that is least demanded.

Pseudo Code - LCH

Input: ρ\rhoρ

  1. Initialize XXX as a random subset of kkk centers from M\mathcal MM.
  2. repeat until no changes occur:
    1. for y∈My\in\mathcal My∈M do
      • Sy←{i∈N:ρ⋅diy<Di(X)}S_y\gets\{i\in N:\rho\cdot d_{iy}\lt D_i(X)\}Sy​←{i∈N:ρ⋅diy​<Di​(X)}
      • if ∣Sy∣≥⌈nk⌉|S_y|\ge\lceil\frac{n}{k}\rceil∣Sy​∣≥⌈kn​⌉ then
        • x∗←arg⁡min⁡x∈X∣{i∈N:dix=Di(X)}∣x^*\gets\arg\min_{x\in X}|\{i\in N:d_{ix}=D_i(X)\}|x∗←argminx∈X​∣{i∈N:dix​=Di​(X)}∣
        • X←(X╲{x∗})∪{y}X\gets(X\diagdown\{x^*\})\cup\{y\}X←(X╲{x∗})∪{y}
  3. return XXX

LCH runs in O~(mn2)\tilde O(mn^2)O~(mn2) time. No guarantee of convergence! aka for a given ρ\rhoρ, there may not exist a ρ\rhoρ-proportional solution.

If LCH terminate, it must return a ρ\rhoρ-PROP solution.

PROP as a Constraint

Problem with LCH and Greedy Capture: They may find a PROP clustering with poor global objective, even when exact PROP clustering with good global objectives exist.

A bad example: k=2k=2k=2, two easily defined clusters containing 40%,60%40\%,60\%40%,60% of the data points respectively. It is possible Greedy Capture only open one point in the larger cluster. It is PROP, but not optimized w.r.t. main object.

So we need to solve this problem by optimizing the kkk-median objective subject to PROP as a constraint.

Consider kkk-median and kkk-means.

🎯Theorem 2. There is a ρ\rhoρ-PROP clustering with kkk-median objective ccc. In polynomial time in mmm and nnn, we can compute a O(ρ)O(\rho)O(ρ)-PROP clustering with kkk-median objective at most 8c8c8c.

We can compute a constant approximate PROP with kkk-median objective at most eight times the minimum kkk-median objective PROP clustering.

Given N,M\mathcal N,\mathcal MN,M, X⊂NX\subset \mathcal NX⊂N with ∣X∣<k|X|\lt k∣X∣<k, find the minimum value of ρ\rhoρ s.t. XXX is ρ\rhoρ-PROP.

Trivial idea: One can solve the audit problem exactly in O((k+m)n)O((k+m)n)O((k+m)n) time by computing for each y∈My\in\mathcal My∈M, the quantity ρy\rho_yρy​, the ⌈nk⌉\lceil\frac{n}{k}\rceil⌈kn​⌉ largest value of Di(X)d(i,y)\frac{D_i(X)}{d(i,y)}d(i,y)Di​(X)​. Then find the yyy that maximizes ρy\rho_yρy​. Again, this takes quadratic time, which can be worse than the time taken to find the clustering itself.

(ϵ,δ)(\epsilon,\delta)(ϵ,δ)-Audit Problem: Find the minimum value of ρ\rhoρ s.t. XXX is ρ\rhoρ-PROP to (1+ϵ)(1+\epsilon)(1+ϵ)-deviations with probability at least 1−δ1-\delta1−δ.
