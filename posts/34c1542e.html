Markov Chains

Definitions

A discrete-time finite Markov chain is a sequence of random variables X=X0,X1,X2,⋯X=X_0,X_1,X_2,\cdotsX=X0​,X1​,X2​,⋯ taking values in a finite set of states {1,2,⋯ ,n}\{1,2,\cdots,n\}{1,2,⋯,n} s.t. ∀t∈N:\forall t\in\mathbb N:∀t∈N:

Pr⁡(Xt+1=j∣Xt=i)=Qi,j,\Pr(X_{t+1}=j|X_t=i)=Q_{i,j},Pr(Xt+1​=j∣Xt​=i)=Qi,j​,

where Q=(Qi,j)Q=(Q_{i,j})Q=(Qi,j​) is an n×nn\times nn×n stochastic transition matrix.

XtX_tXt​ represents the state of the Markov chain at time ttt and the matrix QQQ encodes the state-to-state transition probabilities. The characteristic property of a Markov chain is that the distribution of XtX_tXt​ depends only on Xt−1X_{t-1}Xt−1​. The conditional probability above is independent of the time ttt.

The ttt-step transition probabilities are given by the power matrix QtQ^tQt, i.e.,

Pr⁡(Xs+t=j∣Xs=i)=Qi,jt,\Pr(X_{s+t}=j|X_s=i)=Q_{i,j}^t,Pr(Xs+t​=j∣Xs​=i)=Qi,jt​,

Pr⁡(Xt=j)=∑i=1nQi,jt⋅Pr⁡(X0=i).\Pr(X_t=j)=\sum_{i=1}^nQ_{i,j}^t\cdot\Pr(X_0=i).Pr(Xt​=j)=i=1∑n​Qi,jt​⋅Pr(X0​=i).

A Markov chain is completely determined by the distribution of X0X_0X0​ and the transition matrix QQQ.

A Markov chain can be represented as a weighted directed graph whose vertices are the states of the chain and in which there is an edge (i,j)(i,j)(i,j) of weight Qi,jQ_{i,j}Qi,j​ whenever Qi,j>0Q_{i,j}\gt0Qi,j​>0. We define the weight of a path to the product of the weights of the edges along the path, i.e. the probability to take the path. Then the sum of the weights of all length-kkk paths iii to jjj is Qi,jkQ_{i,j}^kQi,jk​.

The term random walk usually refers to the random process in which a particle moves on a graph, selecting the next node uniformly at random from the adjacent nodes. It is the special case of a Markov chain when for every state iii all probabilities Qi,jQ_{i,j}Qi,j​ for j≠ij\neq ij=i are equal and Qi,i=0Q_{i,i}=0Qi,i​=0.

A lazy random walk is similar but Qi,i=1/2Q_{i,i}=1/2Qi,i​=1/2.

Randomized 2-SAT

Given a 2-CNF formula, either find a satisfying assignment or determine that no such assignment exists.

CNF aka conjunctive normal form，合取范式，形如 ()\and()\and()

2-CNF 就是 ()\and()\and()\cdots\and()，括号内里面是两个布尔变量的析取。

DNF aka disjunctive normal form，析取范式，形如 ()\or()\or()

2-SAT is solvable in polynomial time (in fact linear time).

There is a polynomial time randomized algorithm for 2-CNF formulas whose analysis is based on a random walk.

Pseudo-code

Randomized 2-SAT(φ)(\varphi)(φ)

  • Pick a random assignment
  • Repeat 2n22n^22n2 times or until a satisfying assignment is found
    • Pick an unsatisfied clause CCC
    • Pick a literal in CCC uniformly at random, and flip its value
  • Return current assignment if satisfying, else return unsatisfiable

Analysis

If the φ\varphiφ is not satisfiable then the algorithm returns unsatisfiable. However, if φ\varphiφ is satisfiable then the algorithm might not find a satisfying assignment. We analyze the probability that the algorithm gives correct answer.

Consider φ\varphiφ is satisfiable. Let SSS be a particular assignment that satisfies φ\varphiφ.

Then we bound the probability that algorithm finds SSS. If φ\varphiφ is not satisfied by an assignment AAA there must be an unsatisfied clause CCC. Observe that one or both literals of CCC must have different truth values under AAA and SSS. Thus if we flip a random literal in CCC then the probability that we increase the agreement between AAA and SSS is either 1/21/21/2 or 111.

Randomized 3-SAT

Extending the randomized algorithm for 2-SAT to 3-SAT.

3-SAT is NP-complete. We can use O(2n)O(2^n)O(2n) exhaustive search to solve it.

However, we can improve the performance of this procedure by adapting random restarts into search. A random truth assignment leads to a binomial distribution over the states of the Markov chain XXX centered around the state n/2n/2n/2.

Since Markov chain has a tendency to move towards state 000, after running it for a little while we are better off restarting with a random distribution than continuing to flip literals.

After random restart, by Chernoff bounds, the current assignment differs from SSS is approximately n/2n/2n/2 positions with high probability.

There is an algorithm trying to find a satisfying assignment in N=O((4/3)nn)N=O((4/3)^n\sqrt n)N=O((4/3)nn​) phases. where each phase starts with a random restart and consists of 3n3n3n literal flips.

Stationary Distributions

Definition

Let XXX be a Markov chain with transition matrix QQQ. A probability distribution π\piπ on the set of states is called stationary distribution of XXX if

π=πQ.\pi=\pi Q.π=πQ.

Given states i,ji,ji,j of a Markov chain, the hitting time Hi,jH_{i,j}Hi,j​ is a random variable giving the number of steps to visit state jjj starting from state iii. Formally we have

Hi,j=min⁡{t>0:Xt=j∣X0=i}.H_{i,j}=\min\{t\gt0:X_t=j|X_0=i\}.Hi,j​=min{t>0:Xt​=j∣X0​=i}.

In general hi,jh_{i,j}hi,j​ may be infinite. Certainly hi,j=∞h_{i,j}=\inftyhi,j​=∞ if jjj is not reachable from iii in the underlying graph, or, if iii can reach a state kkk from which jjj is no longer reachable. We therefore restrict attention to Markov chains whose underlying graph consists of a single strongly connected component. We call such chains irreducible.

We can still have hi,j=∞h_{i,j}=\inftyhi,j​=∞ in an irreducible chain.

s-t connectivity

We bound on the cover time of random walks to design an impressively simple space efficient algorithm for the fundamental problem of s-t connectivity. In this problem we are given an undirected graph with nnn nodes, not necessarily connected. We are also given two nodes s,ts,ts,t and we want to find out whether s,ts,ts,t are connected.

Pseudo-code

s-t connectivity(G,s,t)(G,s,t)(G,s,t)

  • start a random walk from sss
  • If ttt is reached in 4n34n^34n3 steps then return reachable else return unreachable

No false positives in the algorithm.

By Markov’s inequality, the algorithm outputs a path from sss to ttt with probability 1/21/21/2.

The algorithm only needs to store a constant number of vertex-addresses at any one time.

If follows that we can decide reach-ability in undirected graph in O(log⁡n)O(\log n)O(logn) space using randomization.
