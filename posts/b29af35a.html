Digest from Reference Paper

Efficient algorithms for the approximate and exact nearest neighbor problem.

Goal: Preprocess a dataset of objects. Later, given a new query object, one can quickly return the dataset object that is most similar to the query.

Nearest Neighbor Problem

Given a collection of nnn points, build a data structure which, given any query point, reports the data point that is closest to the query.

A point Rd\mathbb R^dRd and the distance metric. It’s hard when ddd is large.

In approximation version, the algorithm is allowed to return a point whose distance from the query is at most ccc times the distance from the query to its nearest points.

c>1c\gt1c>1 is called the approximation factor.

One of the most popular algorithms for performing approximate search in high dimensions based on locality-sensitive hashing.

Key idea: hash the point using several hash functions to ensure that for each function, the probability of collision is much higher for objects that are close to each other than for those that are far apart. Then one can determine near neighbors by hashing the query point and retrieving elements stored in buckets containing that point.

Definitions

We say point ppp is an RRR-near neighbor of a point qqq if the distance between ppp and qqq is at most RRR.

Definition of Randomized ccc-approximate RRR-near neighbor (abbr. (c,R)(c,R)(c,R)-NN). Given a set PPP of points in a ddd-dimensional space Rd\mathbb R^dRd, and parameters R>0,δ>0R\gt0,\delta\gt0R>0,δ>0, construct a data structure s.t., given any query point qqq, if there exits an RRR-near neighbor of qqq in PPP, it reports some cRcRcR-near neighbor of qqq in PPP with probability 1−δ1-\delta1−δ.

  • We can assume R=1R=1R=1, just by simply dividing all coordinates by RRR. Thus ccc-NN.

Definition of Randomized RRR-near neighbor reporting. Given a set PPP of points in a ddd-dimensional space Rd\mathbb R^dRd, and parameter R>0,δ>0R\gt0,\delta\gt0R>0,δ>0, construct a data structure that, given any query point qqq, reports each RRR-near neighbor of qqq in PPP with probability 1−δ1-\delta1−δ.

Locality-Sensitive Hash Function

The LSH algorithm relies on the existence of locality-sensitive hash functions.

Let H\mathcal HH be a family of hash functions mapping Rd\mathbb R^dRd to some universe UUU. For any 2 point ppp and qqq, consider a process in which we choose a function hhh from H\mathcal HH uniformly at random, and analyze the probability that h(p)=h(q)h(p)=h(q)h(p)=h(q).

The family H\mathcal HH is called (R,cR,P1,P2)(R,cR,P_1,P_2)(R,cR,P1​,P2​)-sensitive if any two points p,q∈Rdp,q\in\mathbb R^dp,q∈Rd.

  • if ∥p−q∥≤R\Vert p-q\Vert\le R∥p−q∥≤R then Pr⁡H[h(q)=h(p)]≥P1\Pr_\mathcal H[h(q)=h(p)]\ge P_1PrH​[h(q)=h(p)]≥P1​
  • if ∥p−q∥≥cR\Vert p-q\Vert\ge cR∥p−q∥≥cR then Pr⁡H[h(q)=h(p)]≤P2\Pr_\mathcal H[h(q)=h(p)]\le P_2PrH​[h(q)=h(p)]≤P2​

LSH Algorithm

An LSH family H\mathcal HH is used for efficient algorithm for approximate near neighbor search.

For parameters kkk and LLL, we choose LLL functions gj(q)=(h1,j(q),⋯ ,hk,j(q))g_j(q)=(h_{1,j}(q),\cdots,h_{k,j}(q))gj​(q)=(h1,j​(q),⋯,hk,j​(q)), where ht,j(1<t<k,1<j<L)h_{t,j}(1\lt t\lt k,1\lt j\lt L)ht,j​(1<t<k,1<j<L) are chosen independently and uniformly at random from H\mathcal HH.
