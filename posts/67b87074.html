随机算法 8 Faster Dimensionality Reduction

Created2024-03-18|Updated2024-06-10|AUDatalogi随机算法
|Word Count:3.2k|Reading Time:19mins|Post Views:

Lecture notes by Kasper.

This lecture: different approaches used to speed up the JL transform.

Recall: classic JL transform works by drawing a random matrix AAA with coordinates i.i.d. either N(0,1)\mathcal N(0,1)N(0,1) or uniform random among {−1,+1}\{-1,+1\}{−1,+1} and then embedding a vector xxx to m−1/2Axm^{-1/2}Axm−1/2Ax. m=O(ϵ−2log⁡n)m=O(\epsilon^{-2}\log n)m=O(ϵ−2logn) if one wants to preserve all pairwise distances to within (1±ϵ)(1\pm\epsilon)(1±ϵ) and embedding time is O(dm)O(dm)O(dm). The target dimensionality is known to be optimal via a result by Larsen and Nelson.

Sparse Embeddings

Using an embedding matrix AAA that is sparse.

Sparsity

A sparse matrix is one with few non-zeroes.

If each column of AAA has only ttt non-zeroes, then the embedding time improves to O(dt)O(dt)O(dt).

Example of sparse: bag of words (words are embedded to vector.)

With vector representation, documents with similar words map to points that are close. Here vectors are very sparse.

The sparsity of vector xxx: ∣∣x∣∣0=||x||_0=∣∣x∣∣0​= number of non-zeroes in xxx.

The embedding time of classic JL is O(m∣∣x∣∣0)O(m||x||_0)O(m∣∣x∣∣0​) and with a matrix having ttt-sparse columns, it’s O(t∣∣x∣∣0)O(t||x||_0)O(t∣∣x∣∣0​).

Kane and Nelson showed that the following construction with high probability preserves all pairwise distances for a set of nnn points:-

  • For each column of the matrix A∈Rm×dA\in\mathbb R^{m\times d}A∈Rm×d, pick a uniform random set of t=O(ϵ−1log⁡n)t=O(\epsilon^{-1}\log n)t=O(ϵ−1logn) rows and assign the corresponding entries either −1-1−1 or +1+1+1 uniformly at random and independently.
  • Then embed xxx as t−1/2Axt^{-1/2}Axt−1/2Ax. Hence the embedding time compared to standard JL went from O(∣∣x∣∣0ϵ−1log⁡n)O(||x||_0\epsilon^{-1}\log n)O(∣∣x∣∣0​ϵ−1logn), ϵ−1\epsilon^{-1}ϵ−1 improvement.
  • The target dimensionality mmm remains optimal O(ϵ−2log⁡n)O(\epsilon^{-2}\log n)O(ϵ−2logn).

The expected length of the embedded vector is the same as the original vector:

E[∣∣t−1/2Ax∣∣22]=t−1E[∣∣Ax∣∣22]=t−1∑i=1mE[(∑j=1dai,jxj)2]=t−1∑i=1m(∑j=1dE[ai,j2xj2]+∑j=1d∑h≠jE[ai,jai,hxjxh])=t−1∑i=1m∑j=1dE[ai,j2]xj2=t−1∑i=1m∑j=1d(t/m)xj2=∑j=1dxj2=∣∣x∣∣22\mathbb E[||t^{-1/2}Ax||_2^2]=t^{-1}\mathbb E[||Ax||_2^2]\\ =t^{-1}\sum_{i=1}^m\mathbb E[(\sum_{j=1}^da_{i,j}x_j)^2]\\ =t^{-1}\sum_{i=1}^m(\sum_{j=1}^d\mathbb E[a_{i,j}^2x_j^2]+\sum_{j=1}^d\sum_{h\neq j}\mathbb E[a_{i,j}a_{i,h}x_jx_h])\\ =t^{-1}\sum_{i=1}^m\sum_{j=1}^d\mathbb E[a_{i,j}^2]x_j^2\\ =t^{-1}\sum_{i=1}^m\sum_{j=1}^d(t/m)x_j^2\\ =\sum_{j=1}^dx_j^2\\ =||x||_2^2E[∣∣t−1/2Ax∣∣22​]=t−1E[∣∣Ax∣∣22​]=t−1i=1∑m​E[(j=1∑d​ai,j​xj​)2]=t−1i=1∑m​(j=1∑d​E[ai,j2​xj2​]+j=1∑d​h=j∑​E[ai,j​ai,h​xj​xh​])=t−1i=1∑m​j=1∑d​E[ai,j2​]xj2​=t−1i=1∑m​j=1∑d​(t/m)xj2​=j=1∑d​xj2​=∣∣x∣∣22​

In the above, we use E[ai,j]=E[ai,j]E[ai,h]=0\mathbb E[a_{i,j}]=\mathbb E[a_{i,j}]\mathbb E[a_{i,h}]=0E[ai,j​]=E[ai,j​]E[ai,h​]=0 by independence and the fact that ai,ja_{i,j}ai,j​ is symmetric around 0.

Moreover, we use E[ai,j2]=t/m\mathbb E[a_{i,j}^2]=t/mE[ai,j2​]=t/m since ai,ja_{i,j}ai,j​ takes either the value −1-1−1 or +1+1+1 with probability t/mt/mt/m and in both cases, ai,j2a_{i,j}^2ai,j2​ takes value 111.

Nelson and Nguyen proved a lower bound: any sparse embedding matrix AAA must have t=Ω(ϵ−1log⁡n/log⁡(1/ϵ))t=\Omega(\epsilon^{-1}\log n/\log(1/\epsilon))t=Ω(ϵ−1logn/log(1/ϵ)), i.e., the upper bound can be improved by at most a log⁡(1/ϵ)\log(1/\epsilon)log(1/ϵ) factor.

Setting largest possible sparsity remains a fundamental open problem in dimensionality reduction.

Feature Hashing

Another line of work tries to improve the embedding time further by making assumptions on the input data.

Take bag of words as example. If one removes very frequently occurring words, then it seems reasonable to assume that the vector representing a document has no large coordinates. More formally, the ratio between ∣∣x∣∣∞||x||_\infty∣∣x∣∣∞​ and ∣∣x∣∣2||x||_2∣∣x∣∣2​ is small.

Under such assumptions, it’s possible to speed up the sparse transforms from above.

Feature hashing by Weinberger et al. takes this to the extreme by letting AAA have exactly one non-zero per column, chosen at a uniform random row and with a value that is uniform among −1-1−1 and +1+1+1. So feature hashing is really the construction of Kane and Nelson with t=1t=1t=1. Therefore it follows from the above calculations that the expected length of embedded vectors are correct. Feature hashing clearly has the fastest possible embedding time of just O(∣∣x∣∣0)O(||x||_0)O(∣∣x∣∣0​).

Freksen, Kamma and Larsen showed that Feature Hashing into the optimal m=O(ϵ−2log⁡n)m=O(\epsilon^{-2}\log n)m=O(ϵ−2logn) dimensions has the JL guarantees (preserve all distances to within (1±ϵ)(1\pm\epsilon)(1±ϵ)) exactly when

∣∣x∣∣∞∣∣x∣∣2=O(ϵ−1/2⋅min⁡{log⁡(1/ϵ)log⁡n,1log⁡n}).\frac{||x||_\infty}{||x||_2}=O(\epsilon^{-1/2}\cdot\min\{\frac{\log(1/\epsilon)}{\log n},\sqrt{\frac{1}{\log n}}\}).∣∣x∣∣2​∣∣x∣∣∞​​=O(ϵ−1/2⋅min{lognlog(1/ϵ)​,logn1​​}).

And more generally, if one embeds into m≥ϵ−2log⁡nm\ge\epsilon^{-2}\log nm≥ϵ−2logn dimensions, then distances are preserved when

∣∣x∣∣∞∣∣x∣∣2=O(ϵ−1/2⋅min⁡{log⁡(ϵm/log⁡n)log⁡n,log⁡(ϵ2m/log⁡n)log⁡n}).\frac{||x||_\infty}{||x||_2}=O(\epsilon^{-1/2}\cdot\min\{\frac{\log(\epsilon m/\log n)}{\log n},\sqrt{\frac{\log(\epsilon^2 m/\log n)}{\log n}}\}).∣∣x∣∣2​∣∣x∣∣∞​​=O(ϵ−1/2⋅min{lognlog(ϵm/logn)​,lognlog(ϵ2m/logn)​​}).

Fast JL Transform

Instead of using sparse matrices for embedding, another approach is to use matrices for which there are efficient algorithms to compute the product AxAxAx.

The Fast JL transform by Ailon and Chazelle.

First observation is similar to what is used in Feature Hashing: If data vectors have only small coordinates (∣∣x∣∣∞∣∣x∣∣2\frac{||x||_\infty}{||x||_2}∣∣x∣∣2​∣∣x∣∣∞​​ is small), then one can use very sparse matrices for the embedding. The main idea is to multiply xxx with a matrix which ensures that coordinates becomes small, and then use a sparse matrix afterwards.

Walsch-Hadamard Matrix

Assume without loss of generality that ddd is a power of 222. Let HdH_dHd​ be the d×dd\times dd×d Walsch-Hadamard matrix.

HdH_dHd​ can be defined recursively as follows:

Again, all rows of Hˉ2d\bar H_{2d}Hˉ2d​ are orthogonal:

  • For two distinct rows in the top half of the matrix, the orthogonality follows by induction.
  • Similarly for two in the bottom half. For one row in the top half of the matrix and one from the bottom half, orthogonality foolows since the top row vector has the form (v∘v)(v\circ v)(v∘v) and bottom has the form w∘−ww\circ-ww∘−w where ∘\circ∘ denotes concatenation.

Write xxx as (x1x2)(\begin{array}{cc}x_1\\x_2\end{array})(x1​x2​​) where x1x_1x1​ is the first d/2d/2d/2 coordinates of xxx and x2x_2x2​ is the last d/2d/2d/2 coordinates.

Let HHH be the normalized WH matrix, i,e, H=d−1/2HˉdH=d^{-1/2}\bar H_{d}H=d−1/2Hˉd​. Then all rows are still orthogonal and the norm of any row is 111. Hence HHH is an orthogonal matrix and ∣∣Hx∣∣22=∣∣x∣∣22||Hx||_2^2=||x||_2^2∣∣Hx∣∣22​=∣∣x∣∣22​ for all vectors x∈Rdx\in\mathbb R^{d}x∈Rd.

That is, HHH preserves all norms. The construction of Alion and Chazelle is then as follows:

  • Draw a random diagonal matrix D∈Rd×dD\in\mathbb R^{d\times d}D∈Rd×d where each entry is uniform be ±1\pm1±1.

  • Draw a matrix P∈Rm×dP\in\mathbb R^{m\times d}P∈Rm×d with m=O(ϵ−2log⁡n)m=O(\epsilon^{-2}\log n)m=O(ϵ−2logn) s.t. for every entry, with probability 1−q1-q1−q we set the entry to 000, and otherwise, we let the entry be N(0,(mq)−1)\mathcal N(0,(mq)^{-1})N(0,(mq)−1) distributed. The expected number of non-zeroes of PPP is qmdqmdqmd.

  • The embedding of xxx is the PHDxPHDxPHDx.

Computing DxDxDx takes O(d)O(d)O(d) time. Multiplying the result with HHH takes O(dlog⁡d)O(d\log d)O(dlogd) time and multiplying with PPP takes expected O(qmd)O(qmd)O(qmd) time.

Ailon and Chazelle showed that the construction achieves the JL guarantee if one sets q=O(log⁡2n/d)q=O(\log^2n/d)q=O(log2n/d), resulting in an embedding time of O(dlog⁡d+mlog⁡2n)=O(dlog⁡d+ϵ−2log⁡3n)O(d\log d+m\log^2n)=O(d\log d+\epsilon^{-2}\log^3n)O(dlogd+mlog2n)=O(dlogd+ϵ−2log3n).

In the first step, we need to show ∣∣HDx∣∣∞=O(log⁡(nd)/d)||HDx||_\infty=O(\sqrt{\log(nd)/d})∣∣HDx∣∣∞​=O(log(nd)/d​) with probability 1−1/n31-1/n^31−1/n3 for any unit vector x∈Rdx\in\mathbb R^dx∈Rd. That is, HDHDHD ensures that most coordinates of xxx are small.
