Fair AI/ML 4 Endeavor to Bridge Gap between 2 and 2.414

Created2024-10-25|Updated2024-10-25|AUDatalogi算法博弈论AI/ML 公平性
|Word Count:919|Reading Time:5mins|Post Views:

We dedicated to solve the open problem left by Greedy Capture: Is there an algorithm that has better PROP guarantee 2≤ρ≤1+22\le\rho\le1+\sqrt22≤ρ≤1+2​?

My Initial Immature Idea

Do some fine-tuning on results of Greedy Capture.

Pseudo code

  1. X←X\getsX← Greedy Capture(N,M,k)(\mathcal N,\mathcal M,k)(N,M,k)
  2. N←∅N\gets\emptyN←∅ // NNN denotes “free” data points
  3. while ∃\exist∃ 2-approximate Block Coalition
    • Nbc←N_{bc}\getsNbc​← all data points in that 2-approx Block Coalition
    • Xbc←X_{bc}\getsXbc​← all cluster centers associated with any data points in NbcN_{bc}Nbc​
    • Naffect←N_{\text{affect}}\getsNaffect​← all data points associated with any cluster centers in XbcX_{bc}Xbc​
    • N←N∪(Naffect╲Nbc)N\gets N\cup(N_{\text{affect}}\diagdown N_{bc})N←N∪(Naffect​╲Nbc​)
    • x∗←x^*\getsx∗← cluster center of 2-approximate Block Coalition
    • X←X╲XbcX\gets X\diagdown X_{bc}X←X╲Xbc​
    • X←X∪{x∗}X\gets X\cup\{x^*\}X←X∪{x∗}
  4. return XXX

Idea is straightforward, directly eliminate 2-approx Block Coalition.

But how to continue, we need to rerun the Greedy Capture at some point.

Improve the Idea

Instead of using Greedy Capture as black box, we do some modification in it.

“Fine-tuning” Greedy Capture

In the while loop of Greedy Capture, we audit whether there exist 2-approx Block Coalition.

If there is one, we open the corresponding cluster center x∗x^*x∗, and fix all ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉ point in the Block Coalition.

Set the fixed radius rbcr_{bc}rbc​, denoting the distance from cluster center of BC to the ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉-th nearest data point.

Open that cluster center.

Then run the Greedy Capture from the scratch neglecting all fixed point. We don’t synchronously increase rbcr_{bc}rbc​ until δ\deltaδ reaches rbcr_{bc}rbc​.

Also auditing the 2-approximate in that Greedy Capture, if there are block coalition related to any point in cluster x∗x^*x∗, we do the same recursively.

Pseudo Code

Fine-tuning Greedy Capture(N,M,k,rbc,x∗)(\mathcal N,\mathcal M,k,r_{bc},x^*)(N,M,k,rbc​,x∗)

Input: N,M,k,rbc,x∗\mathcal N,\mathcal M,k,r_{bc},x^*N,M,k,rbc​,x∗ // rbcr_{bc}rbc​ denotes the fixed radius for 2-block coalition, x∗x^*x∗ denotes the center of 2-block coalition

  1. X←∅,Δ=(δ1,δ2,⋯ ,δm)←(0,0,⋯ ,0),N←NX\gets\empty,\Delta=(\delta_1,\delta_2,\cdots,\delta_m)\gets(0,0,\cdots,0),N\gets\mathcal NX←∅,Δ=(δ1​,δ2​,⋯,δm​)←(0,0,⋯,0),N←N

  2. X←X∪{x∗},δx∗←rbcX\gets X\cup\{x^*\},\delta_{x^*}\gets r_{bc}X←X∪{x∗},δx∗​←rbc​

  3. while N≠∅N\neq\emptyN=∅ do

    • Smoothly and synchronously increase Δ╲δx∗\Delta\diagdown\delta_{x^*}Δ╲δx∗​

    • If δ\deltaδ for other surpass δx∗\delta_{x^*}δx∗​, increase δx∗\delta_{x^*}δx∗​ also.

    • while ∃x∈X\exist x\in X∃x∈X s.t. ∣B(x,δx)∩N∣≥1|B(x,\delta_x)\cap N|\ge1∣B(x,δx​)∩N∣≥1

      • N←N╲B(x,δx)N\gets N\diagdown B(x,\delta_x)N←N╲B(x,δx​)

    • while ∃x∈M╲X\exist x\in\mathcal M\diagdown X∃x∈M╲X s.t. ∣B(x,δx)∩N∣≥⌈n/k⌉|B(x,\delta_x)\cap N|\ge\lceil n/k\rceil∣B(x,δx​)∩N∣≥⌈n/k⌉ do

      • X←X∪{x}X\gets X\cup\{x\}X←X∪{x}
      • N←N╲B(x,δx)N\gets N\diagdown B(x,\delta_x)N←N╲B(x,δx​)

    • while ∃x′∈M╲X\exist x'\in\mathcal M\diagdown X∃x′∈M╲X s.t. x′x'x′ is the center of a 2-block coalition

      • x∗←x′x^*\gets x'x∗←x′

  4. return XXX

Important: DOES IT WORK?

It is obvious that if the algorithm terminates, then we can find 2-approximate PROP clustering.

Running on the Example

On the example where no matter how the clustering is, the PROP approximation can not be better than 2. The algorithm exactly returns 2-PROP clustering.

On the example in which 1+21+\sqrt21+2​ is the lower bound for Greedy Capture, The algorithm returns exactly PROP clustering.

Two symmetry parts, n=6,m=4,k=3n=6,m=4,k=3n=6,m=4,k=3.

  • Finally, x1x_1x1​ are no longer opened. It is exactly PROP clustering.

Questions

There are a few question need to be answered.

❓How to prove that the algorithm always terminate?

❓Can this algorithm always find the best PROP clustering among all (mk)\left( \begin{array}{c}m\\k\end{array} \right)(mk​) possible clustering?
