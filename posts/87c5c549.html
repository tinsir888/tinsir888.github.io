数据挖掘 3 Densiy-based Clustering

Created2024-02-12|Updated2024-02-19|AUDatalogi数据挖掘
|Word Count:2.9k|Reading Time:18mins|Post Views:

Density-based departs from the rigid geometrical structure of representative-based clustering and defines cluster as high-density regions separated by low density regions.

Density is number of points in a certain region in proportion to the size of the region. Different definition of density, different density-based approaches.

DBSCAN

A density-based clustering algorithm grounded into the intuition that a point in a cluster should be density reachable from any other point in that cluster, i.e., there should be “enough” points around each point in a cluster.

This standard does not include any specific on the shape of the cluster, but only constrains the neighborhood of a point. Thus, DBSCAN can handle clusters with arbitrary shape.

Definitions

Neighborhood: For each point x\mathbf xx, we define the neighborhood the set of points that are at distance at most ϵ\epsilonϵ from x\mathbf xx, formally Nϵ(x)={y∈D∣dist(x,y)≤ϵ}N_\epsilon(\mathbf x)=\{\mathbf y\in\mathcal D|dist(\mathbf x,\mathbf y)\le\epsilon\}Nϵ​(x)={y∈D∣dist(x,y)≤ϵ}.

Additionally, a neighborhood is dense if it contains at least MinPtsMinPtsMinPts points, i.e., ∣Nϵ(x)∣≥MinPts|N_\epsilon(\mathbf x)|\ge MinPts∣Nϵ​(x)∣≥MinPts.

Core object: A point x\mathbf xx whose neighborhood contains at least MinPtsMinPtsMinPts points.

Density-reachable: y\mathbf yy is directly density-reachable from x\mathbf xx within ϵ,MinPts\epsilon,MinPtsϵ,MinPts, if y∈Nϵ(x)\mathbf y\in N_\epsilon(\mathbf x)y∈Nϵ​(x), and x\mathbf xx is a core-object.

A point p\mathbf pp is density reachable from x\mathbf xx if p\mathbf pp is directly density reachable from a point y\mathbf yy in the neighborhood Nϵ(x)N_\epsilon(\mathbf x)Nϵ​(x). Density reachability is the transitive closure of directly density reachability. Note that the density reachability is not symmetric. In particular, since p\mathbf pp might not have MinPtsMinPtsMinPts in its neighborhood, it cannot “reach” x\mathbf xx with a sequence of dense areas.

Density-connected: x\mathbf xx is density-connected to a point y\mathbf yy if there is a point p\mathbf pp such that both x\mathbf xx and y\mathbf yy are density-reachable from p\mathbf pp.

x\mathbf xx and y\mathbf yy are not necessarily density-reachable in this case.

Density-based cluster: a non-empty subset S⊆DS\subseteq\mathcal DS⊆D of the data D\mathcal DD that satisfies both

  • Maximality: If a point x∈S\mathbf x\in Sx∈S and y\mathbf yy is density-reachable from x\mathbf xx, then y∈S\mathbf y\in Sy∈S.
  • Connectivity: Each object in SSS is density-connected to all other objects.

A density-based clustering of a database D\mathcal DD is a partition {C1,⋯ ,Ck;N}\{C_1,\cdots,C_k;N\}{C1​,⋯,Ck​;N} s.t. ⋃i=1kCi∪N=D\bigcup_{i=1}^kC_i\cup N=\mathcal D⋃i=1k​Ci​∪N=D, Ci∩Cj=∅,j>i,i,j∈[1,k]C_i\cap C_j=\empty,j\gt i,i,j\in[1,k]Ci​∩Cj​=∅,j>i,i,j∈[1,k], and Ci∩N=∅C_i\cap N=\emptyCi​∩N=∅, where C1,⋯ ,CkC_1,\cdots,C_kC1​,⋯,Ck​ are the density-based clusters. N=D╲{C1,⋯ ,Ck}N=\mathcal D\diagdown\{C_1,\cdots,C_k\}N=D╲{C1​,⋯,Ck​} is the noise cluster of objects that do not belong to any cluster.

      • if x\mathbf xx is a core project then

      • else

The main theoretical idea behind the DBSCAN algorithm is that each object in a density-based cluster CCC is density-reachable from any of its core objects and nothing else is reachable from the core objects.

The DBSCAN is complete and ensures the two properties of maximality and connectivity. The DBSCAN algorithm is a simple iteration around the objects in the database, starting from a random point and finding all reachable points.

Density-reachable objects are collected by performing successive ϵ\epsilonϵ-neighborhood queries that find the points at distance ϵ\epsilonϵ from another point.

Tuning ϵ\epsilonϵ and MinPtsMinPtsMinPts

Before using DBSCAN, we need to decide on parameters ϵ\epsilonϵ and MinPtsMinPtsMinPts. This choice determines the shape and the number of clusters; low ϵ\epsilonϵ or high MinPtsMinPtsMinPts would find too many small clusters, a high ϵ\epsilonϵ might trivially terminate with a single cluster containing the entire dataset.

To find good ϵ\epsilonϵ and MinPtsMinPtsMinPts, there are some heuristics. For example, one could use the point density of the least dense cluster to determine the parameters. One common heuristic uses the distance from kkk-nearest neighbors.

  • kdist(x)kdist(\mathbf x)kdist(x) is the distance from x\mathbf xx to its kkk-nearest neighbor
  • kdistkdistkdist plot is a chart of the kkk-distances of all points, sorted in decreasing order of distance.

The heuristic works as follows

  • Fix a value for MinPtsMinPtsMinPts. If there is no domain knowledge available, a common choice is 2d−12d-12d−1 where ddd is the dimension.
  • Select a “border object” x\mathbf xx from the MinPtsMinPtsMinPts distance polt, ϵ\epsilonϵ is set to MinPtsdist(x)MinPtsdist(\mathbf x)MinPtsdist(x)

Pros and Cons

👍 Advantages:

  1. Does not require to specify number of clusters.
  2. Performs well with clusters of arbitrary shape.
  3. DBSCAN is robust to outlier and is able to handle them.

👎Disadvantages:

  1. It requires domain knowledge for MinPtsMinPtsMinPts and it is not easy to determine ϵ\epsilonϵ.
  2. If clusters are very different in terms of in-cluster densities, then DBSCAN is not well suited to define clusters as it cannot generalize well to clusters with very different densities.

DENCLUE

DENCLUE detects dense regions without specifically specifying the size of the neighborhood.

Main idea: a generalization of neighborhood density in terms of density distribution.

Density Estimation

To estimate an unknown density function in a dataset.

“Non-parametric”

Density estimation:

  • determines an unknown probability density function
  • it is non-parametric and does not assume a fixed probability model
  • estimates the probability density at each point

Nearest neighbor estimation

Instead of fixing the size hhh of the region, the nearest neighbor estimate fixes the number of points kkk in the neighborhood of a point x\mathbf xx and finds the size of the region.

KNN proceeds:

  1. Fix kkk, the minimum number of points that defines a dense region.
  2. Compute the volume vol(Sd(hx))vol(S_d(h_\mathbf x))vol(Sd​(hx​)) as a function of kkk.

Density Attractors

How can we find dense clusters?

Look at the density function and set a threshold ξ\xiξ i.e. a horizontal line in a density points histogram. However, if so, some points in low-density regions might not be assigned to any cluster.

Density attractor: A point x∗\mathbf x^*x∗ is a density attractor if it is a local maxima of the probability density function fff.

In order to discover these peaks in the density function, we need to compute its gradient.

\nabla\hat f(\mathbf x)={\part\over\part\mathbf x}\hat f(\mathbf x)={1\over nh^d}\sum_{i=1}^n{\part\over\part\mathbf x}K({\mathbf x-\mathbf x_i\over h})

For the Gaussian kernel, the gradient takes a particularly neat form

{\part\over\part\mathbf x}K({\mathbf x-\mathbf x_i\over h})=K({\mathbf x-\mathbf x_i\over h})\cdot({\mathbf x-\mathbf x_i\over h})\cdot({1\over h})

which yields

∇f^(x)=1nhd+2∑i=1nK(x−xih)⋅(xi−x)\nabla\hat f(\mathbf x)={1\over nh^{d+2}}\sum_{i=1}^nK({\mathbf x-\mathbf x_i\over h})\cdot(\mathbf x_i-\mathbf x)∇f^​(x)=nhd+21​i=1∑n​K(hx−xi​​)⋅(xi​−x)

Density-based cluster: A set of CCC of data points from a data set D\mathcal DD is a density-based cluster w.r.t. some threshold ξ\xiξ if there exist a set of attractors x1∗,⋯ ,xk∗\mathbf x_1^*,\cdots,\mathbf x_k^*x1∗​,⋯,xk∗​ if

  • Each point x\mathbf xx in CCC attracted to some attractor xi∗\mathbf x_i^*xi∗​.
  • Each density attractor xi∗\mathbf x_i^*xi∗​ exceeds some density threshold ξ\xiξ, i.e., f^(xi∗)≥ξ\hat f(\mathbf x_i^*)\ge\xif^​(xi∗​)≥ξ.
  • Any two density attractors xi∗,xj∗\mathbf x_i^*,\mathbf x_j^*xi∗​,xj∗​ are density reachable. That is, there exists a path from xi∗\mathbf x_i^*xi∗​ to xj∗\mathbf x_j^*xj∗​ such that for all points y\mathbf yy on the path, f^(y)≥ξ\hat f(\mathbf y)\ge\xif^​(y)≥ξ.

Not all points will be part of a cluster. We treat them as noise.

The DENCLUE algorithm: For each point in the data, it finds a candidate attractor. If the attractor’s density is above the threshold ξ\xiξ, it is added to the set AAA of attractors and the point is added to the set RRR of points attracted by x∗\mathbf x^*x∗. Then all the maximal sets of mutually density reachable attractors computed.

We can use it as iterative rule until convergence. The algorithm runs in O(n2T)\mathcal O(n^2T)O(n2T) where TTT is the number of iterations.

DENCLUE vs DBSCAN

DBSCAN corresponds to a discrete kernel, more efficient bit more rough. ϵ,MinPts\epsilon,MinPtsϵ,MinPts corresponds h,ξh,\xih,ξ.

DENCLUE uses Gaussian kernel density based attractors, which is smooth model and more able to capture underlying data distribution than DBSCAN.

Pros and Cons

👍 Advantages

  • Clusters can arbitrary shape and size, not necessarily be convex hull.
  • Number of clusters is determined automatically.
  • Can separate clusters from surrounding noise.
  • Can be supported by spatial index structures.

👎Disadvantages

  • Input parameters may be difficult to determine
  • Can be sensitive to input parameter setting
