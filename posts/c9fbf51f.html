Hash Functions

Universe UUU. Mapping randomly to a range [m]={0,⋯ ,m−1}[m]=\{0,\cdots,m-1\}[m]={0,⋯,m−1}.

Truly random hash function h:U→[m]h:U\rarr[m]h:U→[m].

hhh is idealized, can not be implemented. To represent a truly random hash functions, we need to store at least ∣U∣log⁡2m|U|\log_2m∣U∣log2​m bits. (too large, impossible!)

Idea: hash functions contain only a small element or seed of randomness so that the hash function is sufficiently random for the desired application. e.g. Prime number ppp, for random hash function h:[p]→[0,⋯ ,p−1]h:[p]\rarr[0,\cdots,p-1]h:[p]→[0,⋯,p−1] could be h(x)=(ax+b)mod  ph(x)=(ax+b)\mod ph(x)=(ax+b)modp where a,ba,ba,b are random numbers that together form the random seed describing the function.

Definition and Properties

Definition 1: A hash function h:U→[m]h:U\rarr[m]h:U→[m] is a random variable in the class of all functions U→[m]U\rarr[m]U→[m], i.e., it consists of a random variable h(x)h(x)h(x) for each x∈Ux\in Ux∈U.

3 things we care about hhh:

  1. Space. The size of the random seed that is necessary to calculate h(x)h(x)h(x) given xxx.
  2. Speed. The time it takes to calculate h(x)h(x)h(x) given xxx.
  3. Properties of random variables.

Universal Hashing

To generate a random hash function h:U→[m]h:U\rarr[m]h:U→[m] from a key universe UUU to a set of hash values [m]={0,⋯ ,m−1}[m]=\{0,\cdots,m-1\}[m]={0,⋯,m−1}. We think of hhh as a random variable following some distribution over functions U→[m]U\rarr[m]U→[m]. We want hhh to be universal which means that for any given distinct keys x,y∈Ux,y\in Ux,y∈U, when hhh is picked at random, we have low collision probability:

Pr⁡h[h(x)=h(y)]≤1m\Pr_h[h(x)=h(y)]\le{1\over m}hPr​[h(x)=h(y)]≤m1​

For many applications, it suffices if for some c=O(1)c=O(1)c=O(1), we have

Pr⁡h[h(x)=h(y)]≤cm\Pr_h[h(x)=h(y)]\le{c\over m}hPr​[h(x)=h(y)]≤mc​

Then hhh is called c-approximately universal.

Strong Universality

Strong universality: For h:[u]→[m]h:[u]\rarr[m]h:[u]→[m], we consider pairwise events of the form that for given distinct keys x,y∈[u]x,y\in[u]x,y∈[u] and possibly non-distinct hash values q,r∈[m]q,r\in[m]q,r∈[m], we have h(x)=qh(x)=qh(x)=q and h(y)=rh(y)=rh(y)=r. We note that if hhh is strongly universal, it is also universal since

\Pr[h(x)=h(y)]=\sum_{q\in[m]}\Pr[h(x)=q\and h(y)=q]=m/m^2=1/m

Observation: An equivalent definition of strong universality is that each key is hashed uniformly into [m][m][m], and that every two distinct keys are hashed independently.

Emphasizing the independence, strong universality is also called 2-independence, as it concerns a pair of two events. As for universality, we may accept some relaxed notion of strong universality e.g. 3-independence, k-independence

c-approximately strongly universality: h:[U]→[m]h:[U]\rarr[m]h:[U]→[m] is c-approximately strongly universal if

  1. hhh is c-approximately uniform, meaning for every x∈Ux\in Ux∈U and for every hash value q∈[m]q\in[m]q∈[m], we have Pr⁡[h(x)=q]≤c/m\Pr[h(x)=q]\le c/mPr[h(x)=q]≤c/m;
  2. Every pair of distinct keys hash independently.

Beyond Strong Universality
