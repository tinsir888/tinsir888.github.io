  2. rankH←1rank_H\gets1rankH​←1; rankT←1rank_T\gets1rankT​←1

      • if h′=hh'=hh′=h then

      • if t′=tt'=tt′=t then

        rankT←rankT+1rank_T\gets rank_T+1rankT​←rankT​+1

Ideas for Parallelization

Idea 1 Parallel Sampling

Sampling the negative sets SHS_HSH​ and STS_TST​ can be done in parallel for both head and tail entities. GPU-based libraries like PyTorch can handle large-scale sampling efficiently on GPUs.

I will think about it.

Idea 2 Rank Update

For each sampled triple xh′r′t′x_{h'r't'}xh′r′t′​, it can be parallelized across all samples in SH∪STS_H\cup S_TSH​∪ST​. For-parallel

GPUs are very efficient at handling this kind of parallel computation, especially with batched matrix operations.
