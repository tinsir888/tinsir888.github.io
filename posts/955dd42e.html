Kd-Trees

Problem set

given a set PPP of nnn points in R2\mathbb R^2R2

store PPP in a data structure s.t. given a query rectangle RRR, we can find the points in RRR efficiently.

idea

  • generalize BST to R2\mathbb R^2R2
  • every node vvv corresponds to a rectangular region SvS_vSv​; the points in the subtree of vvv lie in SvS_vSv​.

Build Kd-Tree

BuildKDTree(P,dP,dP,d)

  1. if P={p}P=\{p\}P={p} then return a leaf node representing ppp.

  2. if ddd is even then

    partition PPP into P1,P2P_1,P_2P1​,P2​ using a vertical line through the point vvv with median x-coordinate

  3. else

    partition PPP into P1,P2P_1,P_2P1​,P2​ using a horizontal line through the point vvv with median y-coordinate

  4. l←l\leftarrowl← BuildKDTree(P1,d+1P_1,d+1P1​,d+1)

  5. r←r\leftarrowr← BuildKDTree(P2,d+1P_2,d+1P2​,d+1)

  6. return a tree with root vvv and left subtree lll and right subtree rrr.

Analysis

running time: O(nlog⁡n)O(n\log n)O(nlogn)

size: O(n)O(n)O(n)

Search in Kd-Tree

SearchKDTree(v,Rv,Rv,R)

  1. if vvv is a leaf then report the point ppp stored at vvv if p∈Rp\in Rp∈R.

  2. else

    ​ if Slc(v)⊆RS_{lc(v)}\sube RSlc(v)​⊆R then ReportSubstree(lc(v)lc(v)lc(v))

    ​ else if Slc(v)S_{lc(v)}Slc(v)​ intersects RRR then SearchKDTree(lc(v),Rlc(v),Rlc(v),R)

    ​ if Src(v)⊆RS_{rc(v)}\sube RSrc(v)​⊆R then ReportSubstree(rc(v)rc(v)rc(v))

    ​ else if Src(v)S_{rc(v)}Src(v)​ intersects RRR then SearchKDTree(rc(v),Rrc(v),Rrc(v),R)

Analysis

query time is proportional to number of regions in SvS_vSv​ intersected by RRR.

rectangles visited by ReportSubtree produce output, so their cost ≤k\le k≤k

We bound number of regions Q(n)Q(n)Q(n) intersected by a vertical line:

assume vvv corresponds to a vertical splitting time, qqq in one region of lc(v)lc(v)lc(v) and rc(v)rc(v)rc(v), this suggests Q(n)=1+Q(n2)Q(n)=1+Q(\frac{n}{2})Q(n)=1+Q(2n​)

Q(n)={O(1),if n=12+2Q(n4),otherwiseQ(n)=\begin{cases} O(1),\text{if }n=1\\ 2+2Q(\frac{n}{4}),\text{otherwise} \end{cases}Q(n)={O(1),if n=12+2Q(4n​),otherwise​

4 regions after two steps. any vertical or horizontal lines, intersects 2 regions

Qt(n)=O(n)Qt(n)=O(\sqrt n)Qt(n)=O(n​)

Theorem

a Kd-Tree uses O(n)O(n)O(n) space, can be built in O(nlog⁡n)O(n\log n)O(nlogn) time, and can report all points in a query rectangle RRR in O(n+k)O(\sqrt n+k)O(n​+k) time.

Windowing Queries

Problem set

given a set S\mathcal SS of nnn disjoint line segments in the plane.

store S\mathcal SS in data structure s.t. given a query rectangle RRR, we can find the segments in S\mathcal SS intersecting RRR efficiently.

The segments that intersect RRR

  1. have an endpoint in RRR

    find them using a range query with RRR on the set of end points

  2. intersect the boundary of RRR

**先考虑一种退化的情况：**what if the disjoint lines are orthogonal?

store S\mathcal SS in a data structure s.t. given a vertical query segment qqq, we can find the segments in S\mathcal SS interesting qqq efficiently.

Interval Stabbing Queries

问题：一些平行的线段和一条直线相交的查询

given a set S\mathcal SS of nnn intervals in R1\mathbb R^1R1

store S\mathcal SS in a data structure s.t. given a query value qqq, we can find the intervals in S\mathcal SS intersecting qqq efficiently.

we store S\mathcal SS in a segment tree (aka interval tree) T\mathcal TT

线段树，太典了

T\mathcal TT is balanced BST on the end points

the root of the tree vvv stores the intervals l(v)l(v)l(v) that contain vvv.

the left subtree lll of vvv stores the intervals that lie completely left of vvv.

the right subtree rrr of vvv stores the intervals that lie completely right of vvv.

store these intervals twice:

  1. sorted on increasing left endpoint
  2. sorted on decreasing right end point

Pseudo code

Query(q,T)(q,T)(q,T)

  • if qqq is left of vvv then

    report intervals from l(v)l(v)l(v) using the list of left-end points, stop at the first interval right of qqq.

    Query(q,l)(q,l)(q,l)

  • else if qqq is right of vvv then

    report intervals from r(v)r(v)r(v) using the list of right-end points, stop at the first interval left of qqq.

    Query(q,r)(q,r)(q,r)

Analysis

space usage: O(n)O(n)O(n)

query time: O(log⁡n+k)O(\log n+k)O(logn+k), kkk is numbers of intervals reported

preprocessing time: O(nlog⁡n)O(n\log n)O(nlogn)

Segment stabbing queries

问题：一些平行的线段和一条线段相交的查询

相当于两个维度上的覆盖问题

space usage: O(n)O(n)O(n)

query time: O(log⁡2n+k)O(\log^2 n+k)O(log2n+k), kkk is numbers of intervals reported

preprocessing time: O(nlog⁡n)O(n\log n)O(nlogn)

Unparalleled stabbing queries

再进一步，如果给出的线段不平行呢？（问题进一步变成：给出一些不平行的线段和一条直线，求和这条直线相交的线段个数）

那么线段树 + 优先搜索树方法就行不通了。

split the problem into elementary intervals in which a vertical line intersects the same segments

storing all segments segments in all elementary intervals uses Θ(n2)\Theta(n^2)Θ(n2) space.

再将这些 elementary segments 投影到一个方向上，变成平行的。Project the segments onto the x-axis, yielding intervals. we build a different data structure for interval stabbing.

问题又转成了 interval stabbing

Store the elementary intervals as leaves in a balanced BST T\mathcal TT.

Every node vvv corresponds to an interval lvl_vlv​, which is the union of the elementary intervals stored in its subtree.

store a canonical subset S(v)⊆SS(v)\sube SS(v)⊆S of intervals s.t. s∈Ss\in Ss∈S if and only if lv⊆sl_v\sube slv​⊆s but parent(v)l⊈sparent(v)_l\not\sube sparent(v)l​⊆s.

这里有点 tricky，为了尽量少的节点存下这些 element intervals，尽量往上面的祖先节点存，aka 节点存储的是包含 element intervals 的最大正规集 maximal canonical set。这样复杂度就是 log⁡n\log nlogn 级别的了（对于每根线段，有 O(log⁡n)O(\log n)O(logn) 个线段树节点存储）

T\mathcal TT is a segment tree.

query: find all nodes vvv s.t. q∈lvq\in l_vq∈lv​, and for each such node report all intervals in S(v)S(v)S(v).

query time: O(log⁡n+k)O(\log n+k)O(logn+k) where kkk is the output size.

space: every interval is stored O(log⁡n)O(\log n)O(logn) times, at most twice per level. O(nlog⁡n)O(n\log n)O(nlogn) in total.

how do we build T\mathcal TT?

build a BST on the elementary intervals, insert the intervals in s∈Ss\in Ss∈S one by one.

to insert sss we visit at most 4 nodes per level.

preprocessing time: O(nlog⁡n)O(n\log n)O(nlogn)

把查询的直线换成线段呢？

space O(nlog⁡n)O(n\log n)O(nlogn)

query O(log⁡2n+k)O(\log^2n+k)O(log2n+k)

preprocessing time O(nlog⁡n)O(n\log n)O(nlogn)

Come back to window queries

the segment that intersect RRR

  1. have an endpoint in RRR

    find them using a range query with RRR on the set of end points

    (O(log⁡2n+k)O(\log^2n+k)O(log2n+k) query, O(nlog⁡n)O(n\log n)O(nlogn) space)

  2. intersect the boundary of RRR

    find them using a segment tree

    (O(log⁡2n+k)O(\log^2n+k)O(log2n+k) query, nlog⁡nn\log nnlogn space)

Theorem

we can solve windowing queries in O(log⁡2n+k)O(\log^2n+k)O(log2n+k) time, using O(nlog⁡n)O(n\log n)O(nlogn) space after O(nlog⁡n)O(n\log n)O(nlogn) preprocessing time.
