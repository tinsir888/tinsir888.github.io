3 Hypothesis

SETH

Recap Strong Exponential Time Hypothesis (SETH). For every ϵ>0\epsilon\gt0ϵ>0 there exists an integer k≥3k\ge3k≥3 s.t. CNF−SATCNF-SATCNF−SAT on formulas with clause size at most kkk and nnn variables can not be solved in O(2(1−ϵ)n)O(2^{(1-\epsilon)n})O(2(1−ϵ)n) time even by a randomized algorithm.

3-SUM

3-SUM Hypothesis. 3−SUM3-SUM3−SUM on integers in {−n4,⋯ ,n4}\{-n^4,\cdots,n^4\}{−n4,⋯,n4} can not be solved in O(n2−ϵ)O(n^{2-\epsilon})O(n2−ϵ) time for any ϵ>0\epsilon\gt0ϵ>0 by a randomized algorithm.

3−SUM3-SUM3−SUM Problem: Given a set SSS of nnn integers from {−nc,⋯ ,nc}\{-n^c,\cdots,n^c\}{−nc,⋯,nc} for some constant ccc, determine whether there are x,y,z∈Sx,y,z\in Sx,y,z∈S s.t. x+y+z=0x+y+z=0x+y+z=0. By a standard hashing trick, assume that c≤3+δc\le3+\deltac≤3+δ for any δ>0\delta\gt0δ>0.

SOTA for 3−SUM3-SUM3−SUM: an algorithm in n2(log⁡log⁡n)O(1)/log⁡2nn^2(\log\log n)^{O(1)}/\log^2nn2(loglogn)O(1)/log2n by Chan et al.

APSP

All-Pairs Shortest Paths (APSP) problem: Given an nnn node graph G=(V,E)G=(V,E)G=(V,E), and integer edge weights w:E→{−M,⋯ ,M}w:E\to\{-M,\cdots,M\}w:E→{−M,⋯,M} for some M=nO(1)M=n^{O(1)}M=nO(1), compute for every u,v∈Vu,v\in Vu,v∈V, the shortest path distance d(u,v)d(u,v)d(u,v) in GGG from uuu to vvv. Assume that GGG does not contain negative weight cycles.

求所有点的最短路径，有经典弗洛伊德算法啊。

Floyd-Warshall algorithm can solve APSP in O(n3)O(n^3)O(n3). One can run Dijkstra’s algorithm from every vertex using some tricks, SOTA is n3/exp⁡(log⁡n)n^3/\exp(\sqrt{\log n})n3/exp(logn​).

APSP Hypothesis. No randomized algorithm can solve APSP in O(n3−ϵ)O(n^{3-\epsilon})O(n3−ϵ) time for ϵ>0\epsilon\gt0ϵ>0 on nnn node graphs with edge weights in {−nc,⋯ ,nc}\{-n^c,\cdots,n^c\}{−nc,⋯,nc} and no negative cycles for large enough ccc.

Fine-grained Reductions

Consider problem AAA with textbook runtime a(n)a(n)a(n) and problem BBB with text book runtime b(n)b(n)b(n). Given a supposed O(b(n)1−ϵ)O(b(n)^{1-\epsilon})O(b(n)1−ϵ) time algorithm for BBB for ϵ>0\epsilon\gt0ϵ>0, we would like to compose it with another algorithm that transforms instances of AAA into instances of BBB, to obtain an algorithm for AAA running in time O(a(n)1−ϵ′)O(a(n)^{1-\epsilon'})O(a(n)1−ϵ′) time for ϵ′>0\epsilon'\gt0ϵ′>0 being a function of ϵ\epsilonϵ.

📖Definition of fine-grained reduction. Assume that AAA and BBB are computational problems and a(n)a(n)a(n) and b(n)b(n)b(n) are their conjectured running time lower bounds, respectively. We say AAA (a,b)(a,b)(a,b)-reduces to BBB (A≤a,bBA\le_{a,b}BA≤a,b​B), if for every ϵ>0\epsilon\gt0ϵ>0, there exists δ>0\delta\gt0δ>0, and an algorithm RRR for AAA that runs in time (a(n))1−δ(a(n))^{1-\delta}(a(n))1−δ on inputs of length nnn, making qqq calls to an oracle for BBB with query lengths n1,⋯ ,nqn_1,\cdots,n_qn1​,⋯,nq​, where

∑i=1q(b(ni))1−ϵ≤(a(n))1−δ.\sum_{i=1}^q(b(n_i))^{1-\epsilon}\le(a(n))^{1-\delta}.i=1∑q​(b(ni​))1−ϵ≤(a(n))1−δ.

If A≤a,bBA\le_{a,b}BA≤a,b​B and B≤a,bAB\le_{a,b}AB≤a,b​A, we say that AAA and BBB are fine-grained equivalent, A≡a,bBA\equiv_{a,b}BA≡a,b​B.

若存在一个能在指数位置上打破 AAA 问题猜想时间复杂度下界的算法 RRR，并且该算法包含若干次调用在指数位置上打破 BBB 问题猜想复杂度下界的算法，则称 AAA 能细粒度归约到 BBB。

被绕晕了没有？

The definition implies that if A≤a,bBA\le_{a,b}BA≤a,b​B and BBB has an algorithm with running time O(b(n))1−ϵO(b(n))^{1-\epsilon}O(b(n))1−ϵ, then AAA can be solved by replacing the oracle calls by corresponding runs of the algorithm, obtaining a runtime of O(a(n)1−δ)O(a(n)^{1-\delta})O(a(n)1−δ) for AAA for some δ>0\delta\gt0δ>0.

If A≡a,bBA\equiv_{a,b}BA≡a,b​B, then we can not be able to improve a(n)a(n)a(n) and b(n)b(n)b(n) for AAA and BBB respectively, is the same.

Hardness from SETH

Orthogonal Vectors (OV) problem requires quadratic time under SETH.

kkk-OV can be solved in O(nkd)O(n^kd)O(nkd) time by exhaustive search, for any k≥2k\ge2k≥2. SOTA: nk−1/Θ(log⁡(d/log⁡n))n^{k-1/\Theta(\log(d/\log n))}nk−1/Θ(log(d/logn)). It seems that nk−o(1)n^{k-o(1)}nk−o(1) is necessary.

kkk-OV Hypothesis: No randomized algorithm can solve kkk-OV on instance of size nnn in nk−ϵdO(1)n^{k-\epsilon}d^{O(1)}nk−ϵdO(1) time for constant ϵ>0\epsilon\gt0ϵ>0.

kOV to CNF-SAT

🎯Theorem of reduction from kkk-OV to CNF−SATCNF-SATCNF−SAT: If kkk-OV on sets with NNN vectors from {0,1}m\{0,1\}^m{0,1}m can be solved in Nk−ϵmO(1)N^{k-\epsilon}m^{O(1)}Nk−ϵmO(1) time for any ϵ>0\epsilon\gt0ϵ>0, then CNT−SATCNT-SATCNT−SAT on nnn variables and mmm clauses can be solved in 2(1−ϵ′)nmO(1)2^{(1-\epsilon')n}m^{O(1)}2(1−ϵ′)nmO(1) time for some ϵ′>0\epsilon'\gt0ϵ′>0 and SETH is false.

Parameterized Complexity

Proposed by Rod Downey, Victoria University, New Zealand

Parameterized complexity:

  1. The input to the problem.
  2. The aspects of the input that constitute the parameter.
  3. The question.

A parameterized language is L⊆Σ∗×Σ∗L\subseteq\Sigma^*\times\Sigma^*L⊆Σ∗×Σ∗ where we refer to the second coordinate as parameter.

For simplicity, we can think of L⊆Σ∗×NL\subseteq\Sigma^*\times \NL⊆Σ∗×N.

📖Definition of FPT. A parameterized language LLL is fixed parameter tractable (FPT), iff there is a computable function fff, a constant ccc, and a deterministic algorithm MMM s.t.

∀x,k:⟨x,k⟩∈L⇔M(x,k) accepts,\forall x,k:\lang x,k\rang\in L\Lrarr M(x,k)\text{ accepts},∀x,k:⟨x,k⟩∈L⇔M(x,k) accepts,

and the running time of M(x,k)M(x,k)M(x,k) is ≤f(k)∣x∣c\le f(k)|x|^c≤f(k)∣x∣c.

It’s not hard to see that f(k)∣x∣c=O(∣x∣c)+f(k)f(k)|x|^c=O(|x|^c)+f(k)f(k)∣x∣c=O(∣x∣c)+f(k) for some computable fff.

Example: a parameterized version of Vertex Cover problem:

  • Instance: A graph G=(V,E)G=(V,E)G=(V,E).
  • Parameter: A positive integer kkk.
  • Question: Does GGG have a vertex cover of size ≤k\le k≤k?

There is an algorithm running in time O(1.2738k+kn)O(1.2738^k+kn)O(1.2738k+kn).

So in this problem, we have f(k)=1.2738kf(k)=1.2738^kf(k)=1.2738k. Introduce O∗O^*O∗ notation which ignores f(k)f(k)f(k) be it additive or multiplicative and is only concerned with the exponential part. Thus the algorithm is O∗(n1)O^*(n^1)O∗(n1).

Some optimized methods.

Kernelization

📖Definition of Kernelization. Let L⊆Σ∗×Σ∗L\subseteq\Sigma^*\times\Sigma^*L⊆Σ∗×Σ∗ be a parameterized language. A reduction to a problem kernel, or kernelization, comprises replacing an instance (I,k)(I,k)(I,k) by a reduced instance (I′,k′)(I',k')(I′,k′), called a problem kernel, such that

  1. k′≤kk'\le kk′≤k
  2. ∣I′∣≤g(k)|I'|\le g(k)∣I′∣≤g(k), for some function ggg depending only on kkk.
  3. (I,k)∈L⇔(I′,k′)∈L(I,k)\in L\Lrarr(I',k')\in L(I,k)∈L⇔(I′,k′)∈L.

The reduction from (I,k)(I,k)(I,k) to (I′,k′)(I',k')(I′,k′) must be computable in time polynomial in ∣I∣+∣k∣|I|+|k|∣I∣+∣k∣.

Reduction rules for a kernel for Vertex Cover:

  1. Remove all isolated vertices.
  2. For any degree one vertex vvv, add its single neighbor uuu to the solution set and remove uuu and all of its incident edges from the graph.
  3. If there is a vertex vvv of degree at least k+1k+1k+1, add vvv to the solution set and remove vvv and all of its neighbors.

After applying these rules, we get to a graph (G′,k′)(G',k')(G′,k′), where no vertex in the reduced graph has degree greater than k′≤kk'\le kk′≤k, or less than two. Then simple combinatorics shows that if such a reduced graph has a size kkk vertex cover, its must have size ≤k2\le k^2≤k2. This is the size k2k^2k2 kernelization.

Lower bounds for some W[1]-hard Problems

Chen et.al.: “Tight lower bounds for certain parameterized NP-hard problems” (original paper), Section 4.

The class SNPSNPSNP contains many NPNPNP-hard problems including qqq-SAT, qqq-Colorability, qqq-Set Cover, Vertex Cover, and Independent Set.

It is commonly believed that it is unlikely that all problems in SNPSNPSNP are solvable in sub-exponential time.

🎯Theorem (Vertex Cover reduction has lower bound). Given an instance (G,k)(G,k)(G,k) of Vertex Cover, there is a polynomial time algorithm which either reports that GGG doesn’t have a vertex cover of size kkk, or produces a subgraph G′G'G′ of GGG with at most 2k′2k'2k′ vertices, where k′≤kk'\le kk′≤k, such that GGG has a vertex cover of size kkk iff G′G'G′ has a vertex cover of size k′k'k′.

🎯Theorem of Independent set poly-time solvable hypothesis. If the parameterized Independent Set problem can be solved in time nO(k)n^{O(k)}nO(k), where nnn is the number of vertices in the graph, then all problems in SNPSNPSNP can be solved in sub-exponential time.

🎯Theorem of clique poly-time solvable hypothesis. If Clique, Weight qqq-SAT, Dominating Set, Dominating Clique, or Graph kkk-Cut can be solved in time nO(k)n^{O(k)}nO(k) then all the problems in SNPSNPSNP can be solved in sub-exponential time, where nnn is the size of the universal set from which the kkk elements are to be selected.
