Definition of Voronoi Diagrams

A set PPP of nnn points p1,⋯ ,pnp_1,\cdots,p_np1​,⋯,pn​

decomposition of R2\mathbb R^2R2 into regions R1,⋯ ,RnR_1,\cdots,R_nR1​,⋯,Rn​.

for any q∈Riq\in R_iq∈Ri​, pip_ipi​ is the closest point

Theorems

Cells of the Voronoi diagram of PPP are convex.

Proof:

Consider pip_ipi​. Draw the bisector of pipj‾,j≠i\overline{p_ip_j},j\neq ipi​pj​​,j=i and let hjh_jhj​ be the half space the contains the side of the bisector that includes pip_ipi​. The cell of pip_ipi​ is the intersection of all hjh_jhj​’s.

If not all points of PPP lie on a line, the Voronoi diagram is made of straight line segments or rays and it is connected.

  • the cells are convex regions from the previous proof
  • the finite cells are polygons
  • if there is a full line lll, then it implies that no bisector intersects lll, which means all points lie on a line.

The size of a planar Voronoi diagram is linear.

Proof:

mmm: number of edges in Voronoi graph, III: number of vertices in Voronoi graph, HHH: size infinite cell

Euler’s Formula: V−E+R=2V-E+R=2V−E+R=2

For infinite edges:

  • put a bounding box，加入边界盒后的点数为 VVV，边数为 EEE，面数为 RRR.
  • R=n+1R=n+1R=n+1 one for each input point + outside
  • all vertices have degree three, except four special corner vertices
  • 2E=3(V−4)+4×22E=3(V-4)+4\times 22E=3(V−4)+4×2
  • E=4+H+m,V=4+H+IE=4+H+m,V=4+H+IE=4+H+m,V=4+H+I

一顿推导，得出：

I=2n−2−H≤2n−5I=2n-2-H\le2n-5I=2n−2−H≤2n−5

m=3n−H−3≤3n−6m=3n-H-3\le3n-6m=3n−H−3≤3n−6

Computing 2D Voronoi Diagrams

idea: sweep line

problem: the next point, can totally change everything above the sweep line.

we cannot assume we have compute the diagram above the sweep line correctly.

idea: define a region above the sweep line that we know is stable. Anything in this region is closer to ppp than any point below the line i.e. cell of ccc above the curve is stable.

sweep line: Y=lY=lY=l

curve: Pi(l)=(X−ai)2+(Y−bi)2=(Y−l)2P_i(l)=(X-a_i)^2+(Y-b_i)^2=(Y-l)^2Pi​(l)=(X−ai​)2+(Y−bi​)2=(Y−l)2

Initially, all the Y-coordinates are the event points.

挨着向下扫描，在扫描线上方的点，两两之间用中垂线 bisector 分隔开，扫描线上方的点和扫描线用抛物线分隔开。这样得到的区域就是稳定的。

Other event points are when parabola segment disappear.

Can be found by checking three adjacent parabola segments.

We need a data structure for inserting and deleting parabola segments i.e., a balanced BST.

O(nlog⁡n)O(n\log n)O(nlogn) running time:

log⁡n\log nlogn pere event points

O(n)O(n)O(n) events points (since size of the diagram is linear)

The Delaunay Triangulation

Let PPP be set of nnn points in R2\mathbb R^2R2, let V(P)\mathcal V(P)V(P) be the Voronoi diagram of PPP.

Let D(P)\mathcal D(P)D(P) be the dual graph of V(P)\mathcal V(P)V(P), D(P)\mathcal D(P)D(P) is the Delaunay graph of PPP. 沃诺依图的对偶图是德劳内三角剖分

Theorem: D(P)\mathcal D(P)D(P) is a plane graph

3D Convex Hull

convex hull ofo nnn points in 3D3D3D

composed of

  • vertices
  • edges
  • faces

Complexity of a 3D CH is linear

  1. 假设边是“线”： 在计算几何的上下文中，这里的“线”指的是 3D 凸包的边。 这些边形成凸包的边界。
  2. 从放置在凸包顶点顶部的光源投射边缘的阴影： 想象一下，您有一个光源直接位于凸包最高点（顶点）上方。 现在，将每条边的阴影投影到凸包下方的平坦表面上。 这类似于物体在光照射下投射的阴影。
  3. 由于凸性，边缘的阴影不会交叉： 因为根据定义，凸包是凸形状，因此当投影到平面上时，其边缘的阴影不会相交或相互交叉。 该特性是船体凸度的直接结果。
  4. 由于我们是从不在凸包外部的点进行投影： 投影是从凸包内部的点（具体来说，在最高顶点的顶部）完成的。 这确保了阴影投射在凸包内而不是外部。

Gift Wrapping for 3DCH

wrap a plane around the convex hull

  1. project the points to 2D, find the 2D CH
  2. an edge of 2D CH is a starting point
  3. perform a DFS on the faces. (the face is marked so we don’t visit it again)

O(nh)O(nh)O(nh) time, nnn is number of points, hhh is output size.

Randomized Incremental Algorithm for 3DCH

  1. randomly shuffle the points
  2. create the convex hull of the first four points
  3. for i=1,⋯ ,ni=1,\cdots,ni=1,⋯,n insert pip_ipi​

adding pip_ipi​:

  1. faces visible from pip_ipi​ should be removed
  2. new faces should be added, touching the boundary of the formerly visible faces

an edges is a boundary edge if out of two faces next to it one is visible but the other is not.

to find the visible faces

  • for every point ppp maintain a list of faces l(p)l(p)l(p) visible from ppp
  • for every face fff maintain a list of points l(f)l(f)l(f) visible from fff
  • for every face f∈l(p)f\in l(p)f∈l(p) place a point and back to the pointer p∈l(f)p\in l(f)p∈l(f).

the points that can see fff, l(f)l(f)l(f) is a subset of points that can see f1f_1f1​ and f2f_2f2​, l(f)⊂l(f1)∪l(f2)l(f)\sub l(f_1)\cup l(f_2)l(f)⊂l(f1​)∪l(f2​)

如果一个点能同时看到两个相邻的面，那么这个点一定在这两个面延长面共同的上方。

if FiF_iFi​ is the list of faces visible from pip_ipi​ then adding pip_ipi​ takes ∑f∈Fi∣l(f)∣\sum_{f\in F_i}|l(f)|∑f∈Fi​​∣l(f)∣ time asymptotically
