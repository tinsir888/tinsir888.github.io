Streaming Algorithms: Frequent Items

Streaming setting: Data stream x1,x2,⋯ ,xnx_1,x_2,\cdots,x_nx1​,x2​,⋯,xn​ with xi∈[m]x_i\in[m]xi​∈[m]. The available memory is O(log⁡cn)O(\log^cn)O(logcn). Sub-linear space.

Algorithm: Approximates frequencies for the top kkk items.

Deterministic Algorithm

To estimate item frequencies fjf_jfj​ within an additive error of n/kn/kn/k using with O(klog⁡n)O(k\log n)O(klogn) memory. （log⁡n\log nlogn 这里指的是内存的比特数量。）

  • Maintain set SSS of kkk counters, initialize to 000. For each element xix_ixi​ in stream
    • If xi∈Sx_i\in Sxi​∈S, increment the counter of xix_ixi​.
    • If xi∉Sx_i\not\in Sxi​∈S, add xix_ixi​ to SSS is space is available, else decrement all counters in SSS.

An item in SSS whose count falls to 000 can be removed, the space requirement for storing kkk counters is klog⁡nk\log nklogn and the update time per item is O(k)O(k)O(k). The algorithm estimates the count of an item as the value of its counter or zero if it has no counter.

🤔Claim 1: The frequency estimate njn_jnj​ produced by the algorithm satisfies fj−n/k≤nj≤fjf_j-n/k\le n_j\le f_jfj​−n/k≤nj​≤fj​.

Clearly, njn_jnj​ is less than the true frequency fjf_jfj​. Differences between fjf_jfj​ and the value of the estimate are caused by one of the two scenarios:

  1. The item j∉Sj\not\in Sj∈S, each counter in SSS gets decremented, this is the case when xjx_jxj​ occurs in the stream but the counter for jjj is not incremented.
  2. The counter for jjj gets decremented due to an element j′j'j′ that is not contained in SSS.

Both scenarios result in kkk counters getting decremented hence they can occur at most n/kn/kn/k times, showing that nj≥fj−n/kn_j\ge f_j-n/knj​≥fj​−n/k.

在所有计数器减少前，至少要添加 kkk 次。对于所有计数器的和，每次减少操作都会减少 kkk。所有计数器的和最多减少 n/kn/kn/k 次。所以单个计数器也最多减少 n/kn/kn/k 次。

也就是当所有 item 出现的频率都一样（都是 n/kn/kn/k）的时候，取等号。
