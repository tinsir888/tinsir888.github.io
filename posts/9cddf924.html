数据挖掘 9 Graph Embedding

Created2024-04-01|Updated2024-03-29|AUDatalogi数据挖掘
|Word Count:2k|Reading Time:12mins|Post Views:

Main problem with graphs: lack of predefined node ordering.

Graph is non-Euclidean: the distance among two nodes does not depend on the coordinates of the nodes.

Origin idea of graph embedding: graph could become a set of multi-dimensional points in which Euclidean distances represent similarities among the nodes.

2 methods of embedding:

  1. Linear embedding: linear transformation of the edges.
  2. Random-walk embedding: preserving the probability that a node is reachable from another node.

Linear Embeddings

Simplest form of graph embeddings.

Let zi∈Rd\mathbf z_i\in\mathbb R^dzi​∈Rd the embedding vector of the node iii.

The similarity between vector zi,zj\mathbf z_i,\mathbf z_jzi​,zj​ is sim(i,j)≈zi⊤⋅zjsim(i,j)\approx\mathbf z_i^\top\cdot\mathbf z_jsim(i,j)≈zi⊤​⋅zj​. If two nodes are “close” under some definition of similarity, they should be also similar in the embedding space.

To define embedding vectors, we need

  1. An encoder from nodes to the embedding space.
  2. A similarity among nodes in the graph.

Then we need to define an objective or loss function, s.t. the embeddings are the parameters of optimization. Depending on the loss function, we can then use the appropriate optimizer to find the embeddings zi\mathbf z_izi​

Shallow encoding. We define the embedding matrix Zn×d\mathbf Z^{n\times d}Zn×d in which row iii is the embedding zi\mathbf z_izi​ of node iii. Given Z\mathbf ZZ we can encode each node in its corresponding embedding by a lookup on the embedding matrix. Let v∈In\mathbf v\in\mathbb I^nv∈In the indicator vector that has a 111 in position iii and 000 otherwise. Then

zi=Z⊤v\mathbf z_i=\mathbf Z^\top\mathbf vzi​=Z⊤v

Node similarity. Intuitively check whether two nodes connected, are far form each other in shortest-path, are reachable from random walk, or simply share any common information.

Adjacency-based Similarity

The simplest: two nodes are similar if they share an edge between them.

sim(i,j)=aijsim(i,j)=a_{ij}sim(i,j)=aij​ implies the dot-product among the embedding vectors should approximate the entries of the adjacency matrix aij≈zi⊤zja_{ij}\approx\mathbf z_i^\top\mathbf z_jaij​≈zi⊤​zj​.

We then find embedding that minimize the distance among the entries aija_{ij}aij​ and the dot-product among the embedding vectors zi⊤zj\mathbf z_i^\top\mathbf z_jzi⊤​zj​:

L=∑(i,j)∈V×V∣zi⊤zj−aij∣2\mathcal L=\sum_{(i,j)\in V\times V}|\mathbf z_i^\top\mathbf z_j-a_{ij}|^2L=(i,j)∈V×V∑​∣zi⊤​zj​−aij​∣2

which corresponds to the minimization of the norm between A\mathbf AA and the matrix ZZ⊤,∣∣ZZ⊤−A∣∣F2\mathbf{ZZ^\top},||\mathbf{ZZ^\top-\mathbf A||}_F^2ZZ⊤,∣∣ZZ⊤−A∣∣F2​.

In order to minimize the objective, we use gradient descent or matrix decomposition (such as SVD or QR-decomposition) on A\mathbf AA.

Pros and Cons

👍

  1. The similarity function is straightforward
  2. Many methods for solving the optimization problem.

👎

  1. O(n2)\mathcal O(n^2)O(n2) running time to consider all pairs, O(∣E∣)\mathcal O(|E|)O(∣E∣) if we sum only over the non-zero entries.
  2. O(dn)\mathcal O(dn)O(dn) parameters, one vector for each node.
  3. Only consider direct and local connections through the adjacency matrix.

Multi-hop Similarity

Connections at distance kkk from a certain node to remedy to the strict requirements of the adjacency-based similarity.

The kkk-th power of the adjacency matrix captures kkk-hop connections. Indeed the position i,ji,ji,j in Ak\mathbf A^kAk represents the number of paths of lengths kkk from node iii to node jjj.

Similarly to the case above loss function minimizes the differences between the entry i,ji,ji,j in the kkk-th power of the adjacency matrix and the dot-product among the embedding vectors of the nodes i,ji,ji,j.

L=∑(i,j)∈V×V∣zi⊤zj−aijk∣2\mathcal L=\sum_{(i,j)\in V\times V}|\mathbf z_i^\top\mathbf z_j-a_{ij}^k|^2L=(i,j)∈V×V∑​∣zi⊤​zj​−aijk​∣2

Similarly, this corresponds to minimizing ∣∣Z⊤Z−Ak∣∣F2||\mathbf{Z^\top Z}-\mathbf A^k||_F^2∣∣Z⊤Z−Ak∣∣F2​. In practice, to avoid numerical explosion, GraRep use a log-transformed probabilistic adjacency matrix

a~ijk=max⁡{log⁡(aij/di∑l∈V(alj/dl))k−α,0}\tilde a_{ij}^k=\max\{\log(\frac{a_{ij}/d_i}{\sum_{l\in V}(a_{lj}/d_l)})^k-\alpha,0\}a~ijk​=max{log(∑l∈V​(alj​/dl​)aij​/di​​)k−α,0}

where ddd is node degree, and α\alphaα is a constant shift. GraRep trains embeddings with different kkk and concatenates the embedding vectors.

A more flexible option to capture multi-hop similarities relies on the overlap between node neighborhoods, using overlap functions such as Jaccard Similarity and Adamic-Adar score which is sij=∑l∈N(i)∩N(j)1log⁡N(l)s_{ij}=\sum_{l\in N(i)\cap N(j)}\frac{1}{\log N(l)}sij​=∑l∈N(i)∩N(j)​logN(l)1​. The loss function becomes

L=∑(i,j)∈V×V∣zi⊤zj−sij∣2\mathcal L=\sum_{(i,j)\in V\times V}|\mathbf z_i^\top\mathbf z_j-s_{ij}|^2L=(i,j)∈V×V∑​∣zi⊤​zj​−sij​∣2

One drawback with Linear embedding is, in their current form, they require to compute the pairwise similarities for all nodes, taking O(n2)\mathcal O(n^2)O(n2) in the worst case. For large graphs, a quadratic complexity is not feasible. Moreover. we need to define the similarity beforehand.

Neural Embeddings

Alternative to the pairwise computation by exploiting the efficiency of probability distributions.

Random-walk based embeddings, non-linear similarity embeddings

Random Walk Embeddings

Let zi⊤zj\mathbf z_i^\top\mathbf z_jzi⊤​zj​ be the probability that node iii and node jjj co-occur in a random walk over the graph. We can estimate the probability of visiting node iii from node jjj using random-walk or methods like Personalized PageRank. If we sample a large enough number of random walks. the process converges to the PageRank.

Random walks are efficient as they do not require to traverse the entire graphs, they can be easily simulated and they converge to probability distributions.

Assume we have a strategy RRR to sample random walks. We can define our embedding algorithm:

  1. Run short random walk starting from each node using strategy RRR.

  2. For each node iii, collect NR(i)N_R(i)NR​(i), the multi-set of nodes visited on random walks starting from iii

  3. Optimize the embeddings according to

    L=∑i∈V∑j∈NR(i)−log⁡(p(j∣zi))\mathcal L=\sum_{i\in V}\sum_{j\in N_R(i)}-\log(p(j|\mathbf z_i))L=i∈V∑​j∈NR​(i)∑​−log(p(j∣zi​))

The optimization is to find embeddings that maximize the likelihood of random walk co-occurrences. We can parametrize p(j∣zi)p(j|\mathbf z_i)p(j∣zi​) by using softmax

p(j∣zi)=exp⁡(zi⊤zj)∑l∈Vexp⁡(zi⊤zl)p(j|\mathbf z_i)=\frac{\exp(\mathbf z_i^\top\mathbf z_j)}{\sum_{l\in V}\exp(\mathbf z_i^\top\mathbf z_l)}p(j∣zi​)=∑l∈V​exp(zi⊤​zl​)exp(zi⊤​zj​)​

The above loss can be optimized using gradient descent. However, the nested sum takes O(n2)\mathcal O(n^2)O(n2), although the real bottleneck is the denominator of the probability, that sums over all nodes. One strategy to reduce the complexity is to employ a negative sampling approach.

Negative sampling treats the problem as a classification problem and splits the fraction into two parts.

log⁡exp⁡(zi⊤zj)∑l∈Vexp⁡(zi⊤zl)∼log⁡(σ(zi⊤zj))−∑l=1klog⁡(zl⊤znl)\log\frac{\exp(\mathbf z_i^\top\mathbf z_j)}{\sum_{l\in V}\exp(\mathbf z_i^\top\mathbf z_l)}\sim\log(\sigma(\mathbf z_i^\top\mathbf z_j))-\sum_{l=1}^k\log(\mathbf z_l^\top\mathbf z_{n_l})log∑l∈V​exp(zi⊤​zl​)exp(zi⊤​zj​)​∼log(σ(zi⊤​zj​))−l=1∑k​log(zl⊤​znl​​)

where nl∼pVn_l\sim p_Vnl​∼pV​ is a node sampled from a negative distribution pVp_VpV​ over the nodes and σ\sigmaσ is the sigmoid function. A typical choice of the negative distribution is to sample negative nodes proportional to the degree of each node. The more we sample, the more robust is the approximation. A higher kkk corresponds to higher prior on negative events.

Pros and Cons

👍

  1. Expressive and non-linear
  2. Scale up with sampling techniques

👎

  1. Similarity is fixed to random walks

General Similarities

Another approach to graph embedding is to generalize the similarity-based approaches to any samplable similarity. An appealing choice for similarity is PPR. For a given restart vector v\mathbf vv for node iii the PPR r\mathbf rr is

ri=(I−αAΔ−1)−1(1−α)v\mathbf r_i=(\mathbf I-\alpha\mathbf{A\Delta}^{-1})^{-1}(1-\alpha)\mathbf vri​=(I−αAΔ−1)−1(1−α)v

from which we obtain the PPR-matrix

S=((1−α)I)(I−αAΔ−1)−1\mathbf S=((1-\alpha)\mathbf I)(\mathbf I-\alpha\mathbf{A\Delta}^{-1})^{-1}S=((1−α)I)(I−αAΔ−1)−1

where each row iii represents the PPR vector ri\mathbf r_iri​.

One choice to embed S\mathbf SS is to use SVD; however this embedding method is again linear. We instead consider each row Si\mathbf S_iSi​ separately as probability distributions to reach any node from node iii. This is the approach of VERSE. VERSE computes the embedding matrix Z\mathbf ZZ to preserve the distribution of the similarities node by node.

VERSE optimize the KL-divergence between the distribution of node iii and node jjj, where the KL-divergence between distribution ppp and qqq is

D(p∣∣q)=∑xp(x)log⁡p(x)q(x),D(p||q)=\sum_xp(x)\log\frac{p(x)}{q(x)},D(p∣∣q)=x∑​p(x)logq(x)p(x)​,

Typically, with KL-divergence the distribution ppp is fixed and given as input, while the distribution qqq is the one to learn. If that is the case, in order to learn ppp, notice that

D(p∣∣q)=∑xp(x)log⁡p(x)−p(x)log⁡q(x)D(p||q)=\sum_xp(x)\log p(x)-p(x)\log q(x)D(p∣∣q)=x∑​p(x)logp(x)−p(x)logq(x)

Entropy of ppp minus Cross entropy

The first term, the entropy of ppp is constant if we minimize for qqq. So it suffices to reduce objective to −∑xp(x)log⁡q(x)-\sum_x p(x)\log q(x)−∑x​p(x)logq(x).

For each node iii we can parametrize q(i)q(i)q(i) as the similarity one-to-all in embedding space given by softmax of the dot-product of the embeddings

sim(i,⋅)=exp⁡(Zzi)∑j=1nexp⁡(zi⊤zj)sim(i,\cdot)=\frac{\exp(\mathbf{Zz}_i)}{\sum_{j=1}^n\exp(\mathbf z_i^\top\mathbf z_j)}sim(i,⋅)=∑j=1n​exp(zi⊤​zj​)exp(Zzi​)​
