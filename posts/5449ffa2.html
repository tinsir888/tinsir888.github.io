Fair AI/ML 8 Split and Merge Algorithm on Average Cost

Created2025-02-18|Updated2025-03-05|AUDatalogi算法博弈论AI/ML 公平性
|Word Count:803|Reading Time:4mins|Post Views:

硕士论文就跟 Non-Centroid Clustering 杠上了。我和导师都希望能

  • 探索 Split and Merge 算法的更多性质
  • 扩展 Split and Merge 算法到更一般的情况

仍然只考虑：

  1. binary distance

    the distance of each pair of data point is either 111 or 222.

  2. non-metric

    no restriction of triangle inequality

  3. asymmetric distance

    dis(a,b)≠dis(b,a)dis(a,b)\neq dis(b,a)dis(a,b)=dis(b,a)

Equivalent Setting

Those three conditions is equivalent to APPROVAL VOTING.

  • Each data point is a voter.
  • Each voter approve a subset of other data points

Cost Definition

The maximum and average costs are therefore defined equivalently and accordingly.

The maximum cost:

It’s not hard to prove that Split and Merge algorithm also computes a clustering in the core with BINARY, ASYMMETRIC, NON-METRIC “distance”.

On Average Cost

🎯Theorem (On Average Cost): Split and Merge algorithm computes a clustering in ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉-core.

Assume there exist a subset VVV of ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉ agents where each agent would improve more than multiplicative factor ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉.

In Split and Merge algorithm, the average cost of data point iii is at most ∣Ci∣−1∣Ci∣\frac{|C_i|-1}{|C_i|}∣Ci​∣∣Ci​∣−1​.

Therefore, for each subset of ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉, there exist one agent does not improve its cost (from 111 to 000). (Consider maximum cost, Split and Merge computes the solution in the core, which means for each subset of ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉, not all agent would be strictly better off.)

Thus, the average cost of this agent is at most 111. Possible improved average cost is at least 1⌈n/k⌉\frac{1}{\lceil n/k\rceil}⌈n/k⌉1​, meaning that the improvement can not be greater than factor of ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉.

Can we improve this ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉ factor? No idea… 🤔

Breakthrough: A Variation of Split and Merge Computes Better Approximation w.r.t. Average Cost

Non-metric, asymmetric and binary distance.

Average cost function.

  1. Construct directed graph.

  2. Split phase: Do iteratively

    • If there exist ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉-clique, put them in one cluster and discard them.

    • else if there exist such a cluster, put then in one cluster and discard them:

      The largest group of points s.t. no points is at distance 111 from ⌈n/k⌉\sqrt{\lceil n/k\rceil}⌈n/k⌉​ other points.

      // This step is as hard as MAXCLIQUE.

  3. Merge phase: Pick the smallest clusters, merge them in order to have at most kkk clusters.

🎯Theorem (⌈n/k⌉\sqrt{\lceil n/k\rceil}⌈n/k⌉​-core w.r.t. average cost): This variation outputs a clustering solution in ⌈n/k⌉\sqrt{\lceil n/k\rceil}⌈n/k⌉​-core.

Assume there exist a subset VVV of ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉ agent improving more than ⌈n/k⌉\sqrt{\lceil n/k\rceil}⌈n/k⌉​. In the variation, average cost of an agent iii is ≤∣Ci∣−1∣Ci∣<1\le\frac{|C_i|-1}{|C_i|}\lt1≤∣Ci​∣∣Ci​∣−1​<1.

For each subset of ⌈n/k⌉\lceil n/k\rceil⌈n/k⌉ agents, there exist one agent having average cost ≤⌈n/k⌉⌈n/k⌉\le\frac{\sqrt{\lceil n/k\rceil}}{\lceil n/k\rceil}≤⌈n/k⌉⌈n/k⌉​​. (By the greedy property of second level of split phase)

The possible improved average cost is at least 1⌈n/k⌉\frac{1}{\lceil n/k\rceil}⌈n/k⌉1​. Meaning that improvement can not be more than ⌈n/k⌉\sqrt{\lceil n/k\rceil}⌈n/k⌉​, this leads to contradiction.
