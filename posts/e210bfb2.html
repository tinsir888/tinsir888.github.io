Review of Count Sketch

This course: extends the results for count sketch algorithm (Lecture 5).

Strict turnstile model, where we maintain estimates of some underlying frequency vector fff. The vector fff has coordinates indexed by integers in [U][U][U] for some universe size UUU.

We receive updates one at a time.

An update: (i,Δ)(i,\Delta)(i,Δ) and has the effect of updating fi←fi+Δf_i\gets f_i+\Deltafi​←fi​+Δ for i∈[U]i\in[U]i∈[U] and Δ∈{x}x=−MM\Delta\in\{x\}_{x=-M}^MΔ∈{x}x=−MM​.

Assume: MMM is bounded by a polynomial in nnn i.e. M=nO(1)M=n^{O(1)}M=nO(1). (Weight change fits in O(log⁡n)O(\log n)O(logn) bits)

We also promise that at any given time, fi≥0∀if_i\ge0\forall ifi​≥0∀i.

In the count sketch, we learned how to use count min sketch to support frequency estimation queries i.e. we have a data structure using O(1ϵlog⁡1δ)O(\frac{1}{\epsilon}\log\frac{1}{\delta})O(ϵ1​logδ1​) space that upon receiving an index i∈[U]i\in[U]i∈[U] as query, can return a value fi^\hat{f_i}fi​^​ that with probability at least 1−δ1-\delta1−δ satisfies fi≤fi^≤fi+ϵ∣∣f∣∣1f_i\le\hat{f_i}\le f_i+\epsilon||f||_1fi​≤fi​^​≤fi​+ϵ∣∣f∣∣1​ where ∣∣f∣∣1=∑i∣fi∣||f||_1=\sum_i|f_i|∣∣f∣∣1​=∑i​∣fi​∣.

Problem Formulation

In many applications, we concern about finding heavy hitters rather than estimating frequency.

An ϵ\epsilonϵ heavy hitter is an index iii s.t. fi≥ϵ∣∣f∣∣1f_i\ge\epsilon||f||_1fi​≥ϵ∣∣f∣∣1​.

We still receive frequency updates. However, there is just one query. To answer a query, we must produce a list L\mathcal LL satisfying the following:

  • All ϵ\epsilonϵ heavy hitters are in L\mathcal LL, i.e., all indices iii with fi≥ϵ∣∣f∣∣1f_i\ge\epsilon||f||_1fi​≥ϵ∣∣f∣∣1​ are in L\mathcal LL.
  • No index iii with fi<ϵ2∣∣f∣∣1f_i\lt\frac{\epsilon}{2}||f||_1fi​<2ϵ​∣∣f∣∣1​ is in L\mathcal LL.

The algorithm is allowed to fail with probability δ\deltaδ.

We have no requirements on indices iii s.t. ϵ2∣∣f∣∣1≤fi<ϵ∣∣f∣∣1\frac{\epsilon}{2}||f||_1\le f_i\lt \epsilon||f||_12ϵ​∣∣f∣∣1​≤fi​<ϵ∣∣f∣∣1​.

Solution of Cormode and Muthukrishannan

For simplicity, assume UUU is a power of 222.

A complete binary tree T\mathcal TT on top of UUU (one leaf per index i∈Ui\in Ui∈U). Such a tree has log⁡U+1\log U+1logU+1 levels and we denote the level of the leaves by level 000 and the root’s level by log⁡u\log ulogu. For a node v∈Tv\in\mathcal Tv∈T, let gvg_vgv​ denote the sum of fjf_jfj​’s over all jjj corresponding to leaves in the sub-tree rooted at vvv.

We think of the list of values gvg_vgv​ for the U/2iU/2^iU/2i nodes at level iii as forming another frequency vector on a universe of size U/2iU/2^iU/2i. We use fif^ifi to denote this vector of frequencies for the iii-th level of the tree.

Obviously fji=∑h=02i−1fj2i+hf_j^i=\sum_{h=0}^{2^i-1}f_{j2^i+h}fji​=∑h=02i−1​fj2i+h​, the frequency for a node is also just the sum of the frequencies of its two children.

We focus on the case where we are aiming at an error probability δ\deltaδ of 1/U1/U1/U, i.e., polynomially small in the universe size. The data structure is as follows:

  • For every level iii of T\mathcal TT, we store a count min sketch of fif^ifi having error probability δ′=1/∣U∣2\delta'=1/|U|^2δ′=1/∣U∣2 and approximation ϵ′=ϵ/2\epsilon'=\epsilon/2ϵ′=ϵ/2. On a update (j,Δ)(j,\Delta)(j,Δ), we conceptually add Δ\DeltaΔ to the frequency of all nodes on the root-to-leaf path leading to the leaf corresponding to jjj.

    This can be done by adding Δ\DeltaΔ to f⌊j/2i⌋if^i_{\lfloor j/2^i\rfloor}f⌊j/2i⌋i​ for i=0,⋯ ,log⁡Ui=0,\cdots,\log Ui=0,⋯,logU. Thus an update makes O(log⁡U)O(\log U)O(logU) updates on count min data structures, each with error probability 1/U21/U^21/U2 thus the total update time is O(log⁡2U)O(\log^2U)O(log2U). similarly, we get a space usage of O(1ϵlog⁡2U)O(\frac{1}{\epsilon}\log^2U)O(ϵ1​log2U).

  • The query algorithm: Start at the root node rrr. When visiting a node vvv at level iii, if i=0i=0i=0 (leaf), add vvv to the output list. Otherwise, vvv is an internal node at some level i>0i\gt0i>0. Query the count min sketch on the vector fi−1f^{i-1}fi−1 to obtain estimates of the frequencies stored at the two children of vvv. Recurse into those children for which the estimate is at least ϵ∣∣f∣∣1\epsilon||f||_1ϵ∣∣f∣∣1​.

    The query algorithm works. First define a node vvv in T\mathcal TT to be heavy if its frequency is at least (ϵ/2)∣∣f∣∣1(\epsilon/2)||f||_1(ϵ/2)∣∣f∣∣1​. Otherwise call it light. Since we are in the strict turnstile model and all frequencies in fff are non-negative, it follows that ∣∣f∣∣1=∣∣f0∣∣1=⋯=∣∣flog⁡U∣∣1||f||_1=||f^0||_1=\cdots=||f^{\log U}||_1∣∣f∣∣1​=∣∣f0∣∣1​=⋯=∣∣flogU∣∣1​. We have some observations as follows.

Observation 1. For any level iii in T\mathcal TT, there are no more than 2/ϵ2/\epsilon2/ϵ heavy nodes.

If we had more than 2/ϵ2/\epsilon2/ϵ heavy nodes at some level iii, then we would have ∣∣fi∣∣1≥(2/ϵ)(ϵ/2)∣∣f∣∣1=∣∣f∣∣1=∣∣fi∣∣1||f^i||_1\ge(2/\epsilon)(\epsilon/2)||f||_1=||f||_1=||f^i||_1∣∣fi∣∣1​≥(2/ϵ)(ϵ/2)∣∣f∣∣1​=∣∣f∣∣1​=∣∣fi∣∣1​, a contradiction.

Observation 2. If iii is the index of an ϵ\epsilonϵ heavy nodes at some level iii, then all the nodes on the root-to-leaf path leading to leaf corresponding to iii are heavy.

Since all frequencies are non-negative, it follows that the frequency of an internal node is always at least as large as any of its children’s.

By observation 2, the list L\mathcal LL produced by the query algorithm always contains all the heavy hitters. Now let iii be an index with fi<(ϵ/2)∣∣f∣∣1f_i\lt(\epsilon/2)||f||_1fi​<(ϵ/2)∣∣f∣∣1​. Note that the leaf vvv corresponding to iii is light. It follows that if iii is added to L\mathcal LL, then the query algorithm visited vvv’s parent and made the estimate of vvv’s frequency that was off by more than (ϵ/2)∣∣f∣∣1=ϵ′∣∣f∣∣1(\epsilon/2)||f||_1=\epsilon'||f||_1(ϵ/2)∣∣f∣∣1​=ϵ′∣∣f∣∣1​. This happens with probability no more than 1/U21/U^21/U2 by our setting of parameters for the count min sketches. We can thus union bound over all light leaves and conclude that none of them are added to L\mathcal LL with probability at least 1−1/U1-1/U1−1/U.

To analyze the query time, observe that if no estimate made by a count min sketch fails, the we only visit heavy nodes. By observation 1, there are O(1ϵlog⁡U)O(\frac{1}{\epsilon}\log U)O(ϵ1​logU) heavy nodes in T\mathcal TT. Thus the query time is O(1ϵlog⁡2U)O(\frac{1}{\epsilon}\log^2U)O(ϵ1​log2U) since we make two queries to count min sketches with error probability 1/U21/U^21/U2 in each.

Note that if we want the query time to be worst case, we can simply abort with an empty output list in case we visit more than (2/ϵ)(log⁡U+1)(2/\epsilon)(\log U+1)(2/ϵ)(logU+1) nodes in T\mathcal TT since this means that at least one light node was visited, and this happens with probability no more than 1/U1/U1/U.

To summarize, we have a data structure with worst case update time O(log⁡2U)O(\log^2U)O(log2U), worst case query time O(1ϵlog⁡2U)O(\frac{1}{\epsilon}\log^2U)O(ϵ1​log2U) and space O(1ϵlog⁡2U)O(\frac{1}{\epsilon}\log^2U)O(ϵ1​log2U).

Solution of Larsen, Nelson, Nguyen and Thorup

This solution improves by a factor log⁡U\log UlogU in all parameters at the cost of moving to expected query time.

The change is simple: Keep the exact same data structure as above, expect that all the count min sketches stored for levels i>0i\gt0i>0 are replaced by count min sketches with error probability 1/41/41/4 instead of 1/U21/U^21/U2. The count min sketch at the leaves still has error probability 1/U21/U^21/U2. The query algorithm and update algorithm remains the same.

The space usage is O(1ϵlog⁡U)O(\frac{1}{\epsilon}\log U)O(ϵ1​logU) as each internal level new uses space only O(1ϵ)O(\frac{1}{\epsilon})O(ϵ1​).

Similarly, the update time drops to O(log⁡U)O(\log U)O(logU) because each internal level costs O(1)O(1)O(1) to update. For the correctness probability, note that the argument from the previous section only argues about queries made to the count min sketch at level 000, thus the correctness probability still holds.

The only thing changed is the query time due to more nodes in T\mathcal TT being visited.

We want to bound the expected number of light nodes visited.

Define a light node vvv its light depth d(v)d(v)d(v) as the distance to its nearest heavy ancestor in T\mathcal TT.

Consider all light nodes at levels i≥2i\ge2i≥2. Let VVV be the set of such nodes. If we visit such a node, we spend O(1)O(1)O(1) time making two queries to a count min sketch. Thus if we define an indicator random variable XvX_vXv​ for each such node v∈Vv\in Vv∈V, taking the value 1 if vvv is visited by the query algorithm, and 0 otherwise, then the expected time spend on visiting all light nodes v∈Vv\in Vv∈V is

O(E[∑v∈VXv])=O(∑v∈VE[Xv])=O(∑v∈VPr⁡[v is visited]).O(\mathbb E[\sum_{v\in V}X_v])=O(\sum_{v\in V}\mathbb E[X_v])=O(\sum_{v\in V}\Pr[v\text{ is visited}]).O(E[v∈V∑​Xv​])=O(v∈V∑​E[Xv​])=O(v∈V∑​Pr[v is visited]).

Thus we need to bound Pr⁡[v is visited]\Pr[v\text{ is visited}]Pr[v is visited]. For this, the trick is to use the light depth.

Let Vi⊆VV_i\subseteq VVi​⊆V be the set of nodes in VVV with light depth iii for i=1,⋯ ,log⁡Ui=1,\cdots,\log Ui=1,⋯,logU. Then we have

  • For vvv to be visited, it must be the case that query made in every single ancestor up to its nearest heavy ancestor failed, i.e. returned an estimate that was off by at least ϵ′∣∣f∣∣1\epsilon'||f||_1ϵ′∣∣f∣∣1​.

  • Since these queries to d(v)=id(v)=id(v)=i independent count min data structures with failure probability 1/41/41/4 each, the probability they all fail is at most 1/4i1/4^i1/4i.

    Hence, we have
