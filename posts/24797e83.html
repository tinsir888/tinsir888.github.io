Brief Introduction

Invertible Bloom Lookup Table (abbr. IBLT)

a fancy hash-table stores key-value pair (x,y)(x,y)(x,y)

space budget: ttt

Guarantee

It “works” when the number of key-value pairs, nnn, in table has n≤tn\le tn≤t.

“temporarily” stops working if n>tn\gt tn>t.

starts working again if nnn drops below ttt.

Operations

(x,y) is key-value pair.

insert(x,y), delete(x,y), get(x), listentries.

Details

Start with table with k≥3k\ge3k≥3 rows, m=2×tm=2\times tm=2×t columns.

In each row, we have a hash function h1,⋯ ,hkh_1,\cdots,h_kh1​,⋯,hk​.

hih_ihi​ is universal.

Each of the entry, called cell, stores values:

  • count
  • key_sum
  • value_sum
