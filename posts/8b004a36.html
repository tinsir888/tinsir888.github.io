Reductions

NP-completeness: by polynomial-time many-one reductions (aka Karp reductions)

Studying completeness of P and NL: by log-space many-one reduction. Also too powerful for completeness inside L.

📖Definition 15 (≤mlog⁡\le_m^{\log}≤mlog​-hard/complete). Let LLL be a language and C\mathcal CC be a class of languages. 📖Definition 16. A language LLL is hard for C\mathcal CC under ⪯\preceq⪯ reductions if for all L′∈CL'\in\mathcal CL′∈C we have L′⪯LL'\preceq LL′⪯L. Complexity class has an important property: the complexity class is closed under reductions.

We say C\mathcal CC is closed w.r.t. ⪯\preceq⪯ reductions, if whenever L′⪯LL'\preceq LL′⪯L and L∈CL\in\mathcal CL∈C, we also have L′∈CL'\in CL′∈C. Proof omitted.

NL-completeness

The basic NL-complete problem is given by connectivity in a directed graph (STCONSTCONSTCON).

STCON

The instance: Directed graph DDD and nodes sss and ttt.

Question: Is there a directed path in DDD from sss to ttt?

🎯Theorem 20. STCONSTCONSTCON is complete for NL.

A non-deterministic log-space Turing machine can guess a path from sss to ttt one node at a time, reusing space.

To show NL-hardness, Let L∈NLL\in NLL∈NL be accepted by non-deterministic log-space Turing machine MMM.

Given input xxx s.t. MMM accepts xxx iff there is a path in the configuration graph of MMM on input xxx from the initial configuration to an accepting configuration.

A log-space reduction from LLL to STCONSTCONSTCON is:

On input xxx we generate all configurations of MMM on input xxx, which then forms the set of nodes together with an additional node ttt. Initial configuration becomes the node sss.

Next generate all arcs between configurations corresponding to a single step of the Turing machine and finally arcs between all accepting configurations to the new node ttt. We then have that the resulting directed graph DDD has a path from sss to ttt iff MMM accepts xxx.

By Savitch’s Theorem and Immerman-Szelepcsényi Theorem, STCON∈DSPACE(log⁡2n)STCON\in DSPACE(\log^2n)STCON∈DSPACE(log2n) and STCON‾∈NL\overline{STCON}\in NLSTCON∈NL, respectively.

Consider so-called succinct versions of complete problems of PPP and NPNPNP.

Let CCC be a Boolean circuit with nnn input gates and one output gate. The truthtable of CCC is the string tt(C)∈{0,1}2ntt(C)\in\{0,1\}^{2^n}tt(C)∈{0,1}2n giving the evaluation of CCC on all possible inputs.

Consider a language L⊆{0,1}∗L\subseteq\{0,1\}^*L⊆{0,1}∗. Then define the succinct version SuccinctLSuccinctLSuccinctL of LLL simply by SuccinctL={C∣tt(C)∈L}SuccinctL=\{C|tt(C)\in L\}SuccinctL={C∣tt(C)∈L}. Note that when L∈PL\in PL∈P we have that SuccinctL∈EXPSuccinctL\in EXPSuccinctL∈EXP simply by decompressing the input CCC to tt(C)tt(C)tt(C) in exponential time and then using the polynomial-time Turing machine to check for membership in LLL.
