How to segment a sequence of points with an efficient algorithms and how to find similar documents in a linear manner.

2 algorithm.

The first is clustering a sequence to form kkk partitions with a dynamic programming.

The second is an approximation that allows to compute pairwise distance avoiding O(n2)\mathcal O(n^2)O(n2). The idea is to use efficient signatures and hash the signatures in bands so as similar items will likely end up in the same bucket.

Sequence Segmentation

A sequence is a vector (x1,⋯ ,xn)∈Rn(x_1,\cdots, x_n)\in\mathbb R^n(x1​,⋯,xn​)∈Rn.

Sequence segmentation: Given a set sequence (x1,⋯ ,xn)(x_1,\cdots,x_n)(x1​,⋯,xn​), find kkk contiguous segments that are as homogeneous as possible.

The sequence segmentation problem requires a notion of homogeneity similar to one of clustering. Points in the segment should be subsequences rendering the problem different from that of clustering.

A sequence is homogeneous if its points have a small deviation from the mean. Let D\mathcal DD be a sequence of nnn ddd-dimensional points.

Definition of kkk-segmentation: A kkk-segmentation SSS is a partition of D\mathcal DD into kkk contiguous segments (s1,⋯ ,sk)(s_1,\cdots,s_k)(s1​,⋯,sk​). Each segment sis_isi​ is represented by a ddd-dimensional mean vector μi=1∣s∣∑x∈sx\mu_i=\frac{1}{|s|}\sum_{\mathbb x\in s}\mathbb xμi​=∣s∣1​∑x∈s​x.

The mean vector has the same role of the centroid for kkk-means clustering. The error E(S)E(S)E(S) of a kkk-segmentation SSS is the error of replacing each individual point with the means. Similar to kkk-means, one of the most common errors is sum of squares error:

E(S)=∑s∈S∑s∈S(x−μs)2E(S)=\sum_{s\in S}\sum_{\mathbb s\in S}(\mathbb x-\mu_s)^2E(S)=s∈S∑​s∈S∑​(x−μs​)2

Problem of k-segmentation problem

Given a sequence D=(x1,⋯ ,xn)\mathcal D=(\mathbb x_1,\cdots,\mathbb x_n)D=(x1​,⋯,xn​) of length nnn, find a set of kkk segments SSS of D\mathcal DD s.t. the SSE error is minimized.

A Dynamic Programming Solution

A dynamic programming solution from Bellman solves the kkk-segmentation problem in polynomial time.

The idea is to build the solution bottom-up solving progressively larger instances of the problem.

DP requires the property of optimal sub-problem that states that the solution on a subset of the input is also optimal.

Denote D[1,i]D[1,i]D[1,i] the sub-sequence x1,⋯ ,xi\mathbb x_1,\cdots,\mathbb x_ix1​,⋯,xi​ with i≤ni\le ni≤n. E(i,s)E(i,s)E(i,s) is the error of the optimal segmentation of the sub-sequence D[1,i]D[1,i]D[1,i] with sss segments with s≤ks\le ks≤k. The dynamic programming algorithm follows the recursion

Complexity

The algorithm needs to fill O(nk)\mathcal O(nk)O(nk) table, each cell computes the minimum, takes O(n)\mathcal O(n)O(n) cells to check, each of it computes the error in O(n)\mathcal O(n)O(n). Thus total complexity of O(n3k)\mathcal O(n^3k)O(n3k) for naive version.

Finding Similar Points Efficiently

Comparing each point in the data with other, raising O(n2)\mathcal O(n^2)O(n2).

Given a dataset D={x1,⋯ ,xn}\mathcal D=\{\mathbb x_1,\cdots,\mathbb x_n\}D={x1​,⋯,xn​} of ddd-dimensional data points, find all the pairs of data point (xi,xj)(\mathbb x_i,\mathbb x_j)(xi​,xj​) s.t. d(xi,xj)≤sd(\mathbb x_i,\mathbb x_j)\le sd(xi​,xj​)≤s for some distance function ddd.

With Locality-Sensitive Hashing, we can get rid of O(n2)\mathcal O(n^2)O(n2).

Definition of Jaccard similarity: of two sets is the size of their intersection sim(C_1,C_2)=|C_1\and C_2|/|C_1\or C_2|. The Jaccard distance is then d(C_1,C_2)=1-|C_1\and C_2|/|C_1\or C_2|.

A large number of documents do not fit into memory and we need a representation of each document that allows for preserving the distances while considerably reducing the memory footprint. In order to achieve such a gold, we use the 3 steps:

  1. Shingling: Convert documents to sets.
  2. Min-hashing: Convert large sets to short signatures, while preserving similarity.
  3. LSH: Focus on pairs of signatures likely to be from similar documents.

Shingling

A kkk-shingle is a set of kkk tokens that appear in the document.

Idea: we return a large number of sparse vector with low similarity.

For instance, the set of 222-shingle for document D1=abcabD_1=abcabD1​=abcab is S(D1)={ab,bc,ca}S(D_1)=\{ab,bc,ca\}S(D1​)={ab,bc,ca}.

Long shingles can be easily compressed by hashing. Documents are compared through the hash values. Finally each document is a binary vector in the space of kkk-shingle in which a shingle is a dimension.

We assume that documents have a large number of common shingles if they are similar. However, kkk is a critical choice to compare documents. A small kkk leads to a large number of similar documents. As rule of thumb k=5,10k=5,10k=5,10 is a reasonable choice for short/large documents.

Shingling doesn’t solve the problem of a large number of similarity computation.

Min-hash

The document is converted into a set of shingles and consequently as a binary vector, where each dimension is a shingle and a 111 in position iii of document jjj indicates that the document contains the shingle iii. This way can we represent our dataset as a matrix where each row represents.

Computing Jaccard similarity in such a matrix relies on simple bit-wise AND, intersection and OR.

Min-Hashing is used for faster comparison. It converts the binary representation into small signatures through hashing. The key idea is to hash each column to a small signatures h(C)h(C)h(C) s.t.

  • h(C)h(C)h(C) is small enough that the signature fits in RAM
  • sim(C1,C2)sim(C_1,C_2)sim(C1​,C2​) is the same as the similarity of signatures h(C1),h(C2)h(C_1),h(C_2)h(C1​),h(C2​).

Definition of min-hash function: The hash function hπ(C)h_\pi(C)hπ​(C) returns the index of the first row in which column CCC has value 111.

hπ(C)=min⁡π(C)h_\pi(C)=\min_\pi(C)hπ​(C)=πmin​(C)

LSH

LSH tackles the problem of the comparison by hashing the signatures in an appropriate manner.

Intuition: use a function f(x,y)f(x,y)f(x,y) to determine whether xxx and yyy is a candidate pair whose similarity must be evaluated. For Min-hash matrices we hash the columns of the signature matrix MMM to different buckets. If two documents hashes into the same buckets, they become a candidate pair with similarity sim≥thresholdsim\ge thresholdsim≥threshold.

LSH works:

  • Pick a similarity threshold sss.
  • The columns of xxx and yyy of MMM are a candidate pair if their signatures agree on at least a fraction sss of their rows. M(i,x)=M(i,y)M(i,x)=M(i,y)M(i,x)=M(i,y) for at least a fraction sss of iii.
  • The idea is to hash columns of signatures matrix MMM several times. Similar columns should be likely to hash to the same bucket with high probability.

LSH partitions MMM’s columns into bbb bands each of rrr rows. For each band, compute a signatures using a hash function with kkk buckets, where kkk is arbitrarily large. The candidate column pairs are those that hash to the same bucket for at least 111 band. Tune bbb and rrr to catch most similar pairs, few non-similar pairs.

How to choose bbb and rrr?

Balances false positives / negatives.

We need to strike a trade-off with the number of bands and the number of rows.

As bbb increases, the method becomes more reliable, but the time increases as well.

If t=sim(C1,C2)t=sim(C_1,C_2)t=sim(C1​,C2​) the probability that all rows in a band are equal is trt^rtr and probability that some row in a band are different is 1−tr1-t^r1−tr.

As such the probability that no band is identical is (1−rt)b(1-r^t)^b(1−rt)b that leads to (1−(1−rt)b)(1-(1-r^t)^b)(1−(1−rt)b) the probability that at least 111 band is identical.

Pros and Cons

👍Finds similar items in linear time

👍Easy and fast to implement

👍Generalizes to other similarity measures
