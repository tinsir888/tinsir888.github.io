随机算法 4 Karger's Min Cut Algorithm

Lecturer this time: Ioannis Caragiannis

Lecture notes is based on Karger’s Min Cut Algorithm by Sanjeev Arora, Sanjoy Dasgupta

Karger’s min cut algorithm and its extension. It’s simple for finding the minimum cut in a graph.

Clustering via Graph Cuts

The minimum cut of an undirected graph G=(V,E)G=(V,E)G=(V,E) is a partition of the nodes into two groups V1,V2V_1,V_2V1​,V2​ (s.t. V=V1∪V2V=V_1\cup V_2V=V1​∪V2​ and V1∩V2=∅V_1\cap V_2=\emptyV1​∩V2​=∅) so that the number of edges between V1V_1V1​ and V2V_2V2​ is minimized.

Variant of cut problem:

  1. Minimum global cut: 将图分成两不连通部分所需的最小割

  2. Minimum s-t cut: 分开源点 s 和汇点 t 所需的最小割

    size of minimum s-t cut = value of maximum s-t flow

    Algorithm: Ford-Fulkerson algorithm (compute then remove augmenting paths in a residual network 计算增广路径), Edmonds-Karp algorithm (O(n∣E∣2)O(n|E|^2)O(n∣E∣2))

一个很自然的计算 minimal global cut 的 deterministic 算法：

  1. 选择一个固定源点 s fix a node s

  2. 将其他 n−1n-1n−1 个点分别当作汇点 t，使用 EK 算法执行 n−1n-1n−1 次，分别计算 s 到这些 t 的最大流 the global should disconnect node s from any other node t.

  3. 返回这些最大流的最小值 return the minimum s-t cut in all these executions

    时间复杂度为 O(n2∣E∣2)O(n^2|E|^2)O(n2∣E∣2) It’s the algorithm we wanna beat today.

    O(n4)≤O(n2∣E∣2)≤O(n6)O(n^4)\le O(n^2|E|^2)\le O(n^6)O(n4)≤O(n2∣E∣2)≤O(n6)

Karger’s Algorithm

Randomized algorithm for finding the minimum cut:

  • Repeat until just two nodes remain:
    • Pick an edge of GGG at random and collapse its two endpoints into a single node.
  • For the two remaining nodes u1,u2u_1,u_2u1​,u2​, set V1={nodes that went into u1}V_1=\{\text{nodes that went into }u_1\}V1​={nodes that went into u1​} and V2={nodes that went into u2}V_2=\{\text{nodes that went into }u_2\}V2​={nodes that went into u2​}.

The algorithm has running time O(n2)O(n^2)O(n2)

  • linear time to update node/edge information after a contraction
  • n−2n-2n−2 contraction in total.

Analysis

Some facts help to analyze the algorithm.

Fact 1

If degree(u)degree(u)degree(u) denotes the number of edges touching node uuu, then

∑u∈Vdegree(u)=2∣E∣\sum_{u\in V}degree(u)=2|E|u∈V∑​degree(u)=2∣E∣

Obviously, each edge contributes twice w.r.t. 2 nodes it connects.

Fact 2

If there are nnn nodes, then the average degree of a node is 2∣E∣n2|E|\over nn2∣E∣​.

A very straightforward conclusion.

When we randomly pick a node XXX at random:

E[degree(X)]=∑u∈VPr⁡(X=u)degree(u)=1n∑u∈Vdegree(u)=2∣E∣n\mathbb E[degree(X)]=\sum_{u\in V}\Pr(X=u)degree(u)={1\over n}\sum_{u\in V}degree(u)=\frac{2|E|}{n}E[degree(X)]=u∈V∑​Pr(X=u)degree(u)=n1​u∈V∑​degree(u)=n2∣E∣​

Fact 3

The size of the minimum cut is at most 2∣E∣n2|E|\over nn2∣E∣​.

Consider the partition of VVV into two pieces, one containing a single node uuu, and the other containing the remaining n−1n-1n−1 nodes. The size of this cut is degree(u)degree(u)degree(u). Since this is a valid cut, the minimum cut cannot be bigger than this. i.e.,

∀u∈V,mincut≤degree(u)⇒mincut≤min⁡u∈V(degree(u))≤avgu∈V(degree(u))=2∣E∣n.\forall u\in V, mincut\le degree(u)\\ \Rarr mincut\le\min_{u\in V}(degree(u))\le\text{avg}_{u\in V}(degree(u))=\frac{2|E|}{n}.∀u∈V,mincut≤degree(u)⇒mincut≤u∈Vmin​(degree(u))≤avgu∈V​(degree(u))=n2∣E∣​.

Fact 4

If an edge is picked at random, the probability that it lies across the minimum cut is at most 2n2\over nn2​.

This is because there are ∣E∣|E|∣E∣ edges to choose from, and at most 2∣E∣n2|E|\over nn2∣E∣​ of them are in the minimum cut.

By facts above, we can easily analyze Karger’s algorithm.

Each time an edge is collapsed, the number of nodes decreases by 111. Therefore,

Karger’s algorithm succeeds with probability p≥2n2p\ge {2\over n^2}p≥n22​. Therefore, it should be run Ω(n2)\Omega(n^2)Ω(n2) times, after which the smallest cut found should be chosen.

Another way to implement Karger’s algorithm:

  • Assign each edge a random weight
  • Run Kruskal’s algorithm to get the minimum spanning tree
  • Break the largest edge in the tree to get the two clusters.

Run Multiple Times to Bound Error Probability

In order to boost the probability of success, we simply run the algorithm ℓ(n2)\ell\left(\begin{array}{c}n\\2\end{array}\right)ℓ(n2​) times.

The probability that at least one run succeeds is at least

1−(1−(n2)−1)ℓ(n2)≥1−e−ℓ.1-\left(1-(\begin{array}{c}n\\2\end{array})^{-1}\right)^{\ell(\begin{array}{c}n\\2\end{array})}\ge 1-e^{-\ell}.1−(1−(n2​)−1)ℓ(n2​)≥1−e−ℓ.

Setting ℓ=cln⁡n\ell=c\ln nℓ=clnn, we have error probability ≤1nc\le{1\over n^c}≤nc1​.

It’s easy to implement Karger’s algorithm so that one run takes O(n2)O(n^2)O(n2) time. Therefore, by running the algorithm O(n2log⁡n)O(n^2\log n)O(n2logn) times, we have an O(n4log⁡n)O(n^4\log n)O(n4logn) time randomized algorithm with error probability 1poly(n)1\over\text{poly}(n)poly(n)1​. Still better than EK algorithm for not very sparse graphs.

Faster Version

Proposed by Karger and Stein.

Key idea

In the initial merging, it’s very unlikely we merged an edge in the mincutmincutmincut. Towards the end of the algorithm, our probability of selecting an edge in the mincutmincutmincut grows.

For a fixed minimum cut δ(S)\delta(S)δ(S), the probability that this cut survives down to ℓ\ellℓ vertices is at least Cℓ2/Cn2C_\ell^2/C_n^2Cℓ2​/Cn2​. Thus, for ℓ=n/2\ell=n/\sqrt2ℓ=n/2​ we have probability ≥12\ge{1\over2}≥21​ of succeeding.

Improved algorithm: From a multi-graph GGG, if GGG has at least 666 vertices, repeat twice:

  1. run the original algorithm down to n2+1{n\over\sqrt2}+12​n​+1 vertices.
  2. recurse on the resulting graph.

Return the minimum of the cuts found in the two recursive calls.
