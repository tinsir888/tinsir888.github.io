Lecturer: Iannis

Goal of dimensionality reduction: represent a high dimensional data set in a lower dimensional space while preserving much of the important structure.

PCA (principle component analysis): to maintain as much variance in the data as possible while reducing the dimensionality.

The mapping of data from a high dimensional space to a low dimensional space is called an embedding.

Johnson-Lindenstrauss Dimensionality Reduction

Lecture notes by Allan Grønlund

Main results of JL: any mmm point subset of Euclidean space can be linearly embedded in k=O(log⁡m/ϵ2)k=O(\log m/\epsilon^2)k=O(logm/ϵ2) dimensions without distorting the distance between any pair of points by more than a factor of (1±ϵ)(1\pm\epsilon)(1±ϵ), for any 0<ϵ<120\lt\epsilon\lt\frac{1}{2}0<ϵ<21​.

For PCA to be relevant the original data points x1,⋯ ,xmx_1,\cdots,x_mx1​,⋯,xm​ must be inherently low dimensional.

While JL theorem requires no assumption on the original data and the target dimension is even independent of the input dimension.

Further results: JL lemma is tight even for non-linear embeddings.

Remarks

JL Lemma generalizes to the case where the entries in AAA are instead uniform random signs ±1\pm1±1. Also, it can be shown that the matrix can be made sparse and the amount of randomness required reduced, all steps to make the embedding more efficient.

We can reduce the dimensionality of any dataset by using a simple randomized linear transformation.

But what if we allow non-linear embedding? Amazed: We can’t!

JL embedding lemma is tight even for non-linear embeddings by showing that there exists data set s.t. there is no way to embed into a space with lower dimension than what is promised in the JL Lemma without violating some of the pairwise distances.

Application on K-Means

lecture notes by Kasper

K-Means

To compute kkk centers minimizing the sum of distances each point to its nearest cluster center.

Computing the optimal k-means is NP-hard. So we use heuristic algorithm such as Lloyd’s algorithm.

The Lloyd’s algorithm starts by partitioning the input points into kkk clusters X1,⋯ ,XkX_1,\cdots,X_kX1​,⋯,Xk​ in some way. Repeat: compute the optimal cluster centers, then reassign each point to the cluster center and repeat until no clusters change.

This algorithm may get stuck in a local minimum and depend on initialization.

Let ttt denote the number of iterations, the total running time is O(tndk)O(tndk)O(tndk)

  • ttt iterations
  • compute new clusters in O(nd)O(nd)O(nd) time
  • reassign points in O(ndk)O(ndk)O(ndk) time.

Applying Johnson-Lindenstrauss

To speed up Lloyd’s algorithm

Before running Lloyd’s algorithm, perform a JL transform s.t. all distances between points are preserved to within (1±ϵ)(1\pm\epsilon)(1±ϵ).

Let A∈Rm×dA\in\mathbb R^{m\times d}A∈Rm×d be the embedding matrix s.t. each original point xix_ixi​ is mapped to AxiAx_iAxi​. The new dimensionality is m=O(ϵ−2log⁡n)m=O(\epsilon^{-2}\log n)m=O(ϵ−2logn). Performing the transform takes O(ndϵ−2log⁡n)O(nd\epsilon^{-2}\log n)O(ndϵ−2logn) time, which may be much less than O(tdnk)O(tdnk)O(tdnk). Therefore, running Lloyd’s in the lower dimensional space takes tnkϵ−2log⁡ntnk\epsilon^{-2}\log ntnkϵ−2logn for a total running time of O(nϵ−2(d+kt)log⁡n)O(n\epsilon^{-2}(d+kt)\log n)O(nϵ−2(d+kt)logn).
